## Metadata

| Item | Value |
|---|---|
| Contract version | `2.6.0` |
| Runtime sentinel | `v1` |

## Runtime Status Legend

| Status | Meaning |
|---|---|
| `runtime` | Shipped parser/runtime surface |
| `preview` | Parsed and previewed, but runtime enforcement is partial or host-enforced |
| `generated` | Produced from the registry or another source of truth |
| `compat` | Backward-compatibility shim or alias |

## Compatibility Policy

- Keep `ZYAL_CONTRACT_VERSION` stable for docs-only or example-only changes.
- Add new schema keys only with an explicit contract bump and matching parser/runtime work.
- Unknown keys are rejected before schema decode so drift is loud.
- `VERSION.md` and `CHANGELOG.md` remain metadata; this file is the canonical schema reference.

## Release Checklist

- Regenerate this file with `bun --cwd packages/jekko ./script/generate-zyal-spec.ts --write`.
- Run `bun --cwd packages/jekko test src/agent-script/schema-spec.test.ts src/agent-script/parser.test.ts`.
- Run the repo-level ZYAL proof lanes called out in `agent/test-map.json`.
- Confirm the docs example table and tracked `.zyal` files still parse.
