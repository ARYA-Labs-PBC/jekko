Absolutely — **Ratatui should be the “ultimate version” path**.

The important distinction is this: the uploaded TUI is currently **TypeScript + SolidJS + OpenTUI**, not Rust. So Ratatui cannot be sprinkled into the existing frontend as a normal dependency. To leverage Ratatui properly, the right move is to create a **new Rust TUI frontend** that speaks to the same backend/API/event system already documented in `TUI_BACKEND_API.md`.

That is actually a great architecture fit. The existing backend already exposes REST endpoints, SSE events, session APIs, prompt APIs, provider/config APIs, permission/question APIs, and TUI control events. So the Rust/Ratatui version should not rewrite the backend. It should replace the rendering/input layer.

Ratatui is a strong match for this because it provides immediate-mode Rust rendering, responsive layouts, rich widgets such as charts, sparklines, tables, gauges, scrollable lists, and progress bars, plus dynamic constraint-based layouts that adapt from small tmux panes to large terminals. ([Ratatui][1]) Ratatui also supports terminal backends including Crossterm, Termion, Termwiz, and a `TestBackend` that is useful for unit-testing UI rendering. ([Ratatui][2]) In Ratatui `0.30.0`, the project was modularized into crates such as `ratatui-core`, `ratatui-widgets`, and backend crates, with a newer `ratatui::run()` API and major widget/layout improvements. ([Ratatui][3])

# Ratatui Replatform Addendum: Build the World-Class Rust TUI

## 1. Recommended Strategy

Do **not** port the current TypeScript UI one component at a time.

Instead, build a new Rust frontend:

```txt
Current backend / worker / HTTP / SSE
        │
        │ REST + SSE
        ▼
New Rust Ratatui frontend
```

The current OpenTUI frontend can stay during transition, but the future flagship TUI should be:

```txt
├── Tokio async runtime
├── reqwest HTTP client

└── Atlas visual system
```

This lets you keep the backend strength while making the visible app completely unrecognizable from OpenCode/OpenTUI.

Ratatui gives you a more native, more controllable, more premium Rust TUI surface.

Use it for:

   * Context usage gauges.
   * Agent latency sparklines.

# 5. Runtime Architecture

The Rust TUI should have three concurrent input streams:

```txt

│                 Rust App State               │
├──────────────────────┬───────────────────────┤
```

# 7. Backend Client

Use the existing backend API instead of inventing a new one.

Example `src/backend/client.rs`:

```rust
use anyhow::Result;
use reqwest::Client as HttpClient;
use serde::{Deserialize, Serialize};

#[derive(Clone)]
pub struct Client {
    http: HttpClient,
    base_url: String,
}

impl Client {
    pub fn new(base_url: String) -> Self {
        Self {
            http: HttpClient::new(),
            base_url,
        }
    }

    pub async fn list_sessions(&self) -> Result<Vec<SessionSummary>> {
        let url = format!("{}/session", self.base_url);
        let response = self.http.get(url).send().await?;
        Ok(response.json().await?)
    }

    pub async fn submit_prompt(
        &self,
        session_id: &str,
        request: PromptRequest,
    ) -> Result<()> {
        let url = format!("{}/session/{}/prompt", self.base_url, session_id);
        self.http.post(url).json(&request).send().await?;
        Ok(())
    }

    pub async fn abort_session(&self, session_id: &str) -> Result<()> {
        let url = format!("{}/session/{}/abort", self.base_url, session_id);
        self.http.post(url).send().await?;
        Ok(())
    }

    pub async fn open_theme_picker(&self) -> Result<()> {
        let url = format!("{}/tui/open-themes", self.base_url);
        self.http.post(url).send().await?;
        Ok(())
    }

    pub async fn toggle_theme_mode(&self) -> Result<()> {
        let url = format!("{}/tui/toggle-theme-mode", self.base_url);
        self.http.post(url).send().await?;
        Ok(())
    }
}

#[derive(Debug, Clone, Deserialize)]
pub struct SessionSummary {
    pub id: String,
    pub title: Option<String>,
}

#[derive(Debug, Clone, Serialize)]
pub struct PromptRequest {
    pub text: String,
    pub files: Vec<String>,
}
```

This keeps the Ratatui frontend aligned with the `TUI_BACKEND_API.md` flow.

# 8. SSE Event Stream

Example `src/backend/event_stream.rs`:

```rust
use anyhow::Result;
use eventsource_stream::Eventsource;
use futures_util::StreamExt;
use reqwest::Client;
use tokio::sync::mpsc;

use crate::action::Action;

pub async fn spawn_event_stream(
    base_url: String,
    tx: mpsc::UnboundedSender<Action>,
) -> Result<()> {
    let url = format!("{}/event", base_url);

    tokio::spawn(async move {
        let client = Client::new();

        loop {
            let result = client.get(&url).send().await;

            let Ok(response) = result else {
                let _ = tx.send(Action::BackendError("failed to connect to event stream".into()));
                tokio::time::sleep(std::time::Duration::from_secs(2)).await;
                continue;
            };

            let mut stream = response.bytes_stream().eventsource();

            while let Some(event) = stream.next().await {
                match event {
                    Ok(event) => {
                        match serde_json::from_str::<crate::backend::schema::BackendEvent>(&event.data) {
                            Ok(parsed) => {
                                let _ = tx.send(Action::BackendEvent(parsed));
                            }
                            Err(err) => {
                                let _ = tx.send(Action::BackendError(format!(
                                    "invalid backend event: {err}"
                                )));
                            }
                        }
                    }
                    Err(err) => {
                        let _ = tx.send(Action::BackendError(format!(
                            "event stream error: {err}"
                        )));
                        break;
                    }
                }
            }

            tokio::time::sleep(std::time::Duration::from_millis(500)).await;
        }
    });

    Ok(())
}
```

# 9. Atlas Layout in Ratatui

Example `src/ui/layout.rs`:

```rust
use ratatui::layout::{Constraint, Direction, Layout, Rect};

#[derive(Debug, Clone, Copy)]
pub struct AtlasLayout {
    pub control_strip: Rect,
    pub atlas_rail: Rect,
    pub signal_stream: Rect,
    pub command_runway: Rect,
}

pub fn compute(area: Rect, rail_visible: bool, rail_width: u16) -> AtlasLayout {
    let vertical = Layout::default()
        .direction(Direction::Vertical)
        .constraints([
            Constraint::Length(1), // Control Strip
            Constraint::Min(8),    // Main body
            Constraint::Length(5), // Command Runway
        ])
        .split(area);

    let body = vertical[1];

    let horizontal = if rail_visible {
        Layout::default()
            .direction(Direction::Horizontal)
            .constraints([
                Constraint::Length(rail_width), // Left Atlas Rail
                Constraint::Min(40),            // Signal Stream
            ])
            .split(body)
    } else {
        Layout::default()
            .direction(Direction::Horizontal)
            .constraints([Constraint::Length(0), Constraint::Min(40)])
            .split(body)
    };

    AtlasLayout {
        control_strip: vertical[0],
        atlas_rail: horizontal[0],
        signal_stream: horizontal[1],
        command_runway: vertical[2],
    }
}
```

This locks in the requested left-panel design.

# 15. Keyboard Handling: White/Dark Shortcut

Example `src/keymap.rs`:

```rust
use crossterm::event::{KeyCode, KeyEvent, KeyModifiers};

use crate::action::Action;

#[derive(Debug, Default)]
pub struct KeyState {
    leader_active: bool,
}

impl KeyState {
    pub fn handle(&mut self, key: KeyEvent) -> Option<Action> {
        if key.modifiers.contains(KeyModifiers::CONTROL) && key.code == KeyCode::Char('x') {
            self.leader_active = true;
            return None;
        }

        if self.leader_active {
            self.leader_active = false;

            return match key.code {
                KeyCode::Char('d') | KeyCode::Char('D') => Some(Action::ToggleThemeMode),
                KeyCode::Char('b') | KeyCode::Char('B') => Some(Action::ToggleRail),
                KeyCode::Char('p') | KeyCode::Char('P') => Some(Action::OpenCommandLens),
                _ => None,
            };
        }

        match key.code {
            KeyCode::Esc => Some(Action::SessionAbort),
            KeyCode::Enter => Some(Action::PromptSubmit),
            KeyCode::Backspace => Some(Action::PromptBackspace),
            KeyCode::Char(c) => Some(Action::PromptInput(c)),
            _ => None,
        }
    }
}
```

Behavior:

```txt
Ctrl+X D    Toggle Atlas Dark / Atlas Light
Ctrl+X B    Toggle Atlas Rail
Ctrl+P      Command Lens
Esc         Stop current agent response
Enter       Submit prompt
```

# 16. Render Loop

Example `src/app.rs`:

```rust
use std::time::Duration;

use color_eyre::Result;
use crossterm::event::{self, Event};
use ratatui::DefaultTerminal;
use tokio::sync::mpsc;

use crate::{
    action::Action,
    backend::Client,
    config::Config,
    keymap::KeyState,
    state::AppState,
    ui,
};

pub struct App {
    config: Config,
    backend: Client,
    state: AppState,
    key_state: KeyState,
    tx: mpsc::UnboundedSender<Action>,
    rx: mpsc::UnboundedReceiver<Action>,
}

impl App {
    pub async fn new(config: Config, backend: Client) -> Result<Self> {
        let (tx, rx) = mpsc::unbounded_channel();

        crate::backend::event_stream::spawn_event_stream(
            config.backend_url.clone(),
            tx.clone(),
        )
        .await?;

        Ok(Self {
            config,
            backend,
            state: AppState::default(),
            key_state: KeyState::default(),
            tx,
            rx,
        })
    }

    pub fn run(&mut self, terminal: &mut DefaultTerminal) -> Result<()> {
        loop {
            terminal.draw(|frame| {
                ui::render(frame, &self.state);
            })?;

            while let Ok(action) = self.rx.try_recv() {
                self.update(action)?;
            }

            if event::poll(Duration::from_millis(16))? {
                match event::read()? {
                    Event::Key(key) => {
                        if let Some(action) = self.key_state.handle(key) {
                            self.update(action)?;
                        }
                    }
                    Event::Resize(width, height) => {
                        self.update(Action::TerminalResize { width, height })?;
                    }
                    _ => {}
                }
            }

            if self.state.should_quit {
                break;
            }
        }

        Ok(())
    }

    fn update(&mut self, action: Action) -> Result<()> {
        match action {
            Action::Quit => self.state.should_quit = true,

            Action::ToggleThemeMode => {
                self.state.toggle_theme_mode();
            }

            Action::ToggleRail => {
                self.state.rail_visible = !self.state.rail_visible;
            }

            Action::PromptInput(c) => {
                self.state.prompt.text.push(c);
            }

            Action::PromptBackspace => {
                self.state.prompt.text.pop();
            }

            Action::PromptSubmit => {
                let text = self.state.prompt.text.clone();
                self.state.prompt.text.clear();

                let backend = self.backend.clone();
                let session_id = self.state.active_session_id.clone();

                tokio::spawn(async move {
                    let _ = backend.submit_prompt(
                        &session_id,
                        crate::backend::client::PromptRequest {
                            text,
                            files: vec![],
                        },
                    ).await;
                });
            }

            Action::BackendEvent(event) => {
                self.state.apply_backend_event(event);
            }

            Action::BackendError(message) => {
                self.state.push_error(message);
            }

            _ => {}
        }

        Ok(())
    }
}
```

# 17. Main UI Renderer

Example `src/ui/mod.rs`:

```rust
use ratatui::Frame;

use crate::state::AppState;

pub mod layout;
pub mod style;
pub mod theme;
pub mod widgets;

use widgets::{
    atlas_rail::AtlasRail,
    command_runway::CommandRunway,
    control_strip::ControlStrip,
    signal_stream::SignalStream,
};

pub fn render(frame: &mut Frame, state: &AppState) {
    let palette = theme::atlas(state.theme_mode);
    let area = frame.area();

    frame
        .buffer_mut()
        .set_style(area, ratatui::style::Style::default().bg(palette.background));

    let layout = layout::compute(area, state.rail_visible, state.rail_width);

    frame.render_widget(
        ControlStrip { state, palette },
        layout.control_strip,
    );

    if state.rail_visible {
        frame.render_widget(
            AtlasRail { state, palette },
            layout.atlas_rail,
        );
    }

    frame.render_widget(
        SignalStream { state, palette },
        layout.signal_stream,
    );

    frame.render_widget(
        CommandRunway { state, palette },
        layout.command_runway,
    );

    if state.command_lens_open {
        let area = centered_rect(70, 60, area);
        frame.render_widget(
            widgets::command_lens::CommandLens { state, palette },
            area,
        );
    }
}

fn centered_rect(percent_x: u16, percent_y: u16, area: ratatui::layout::Rect) -> ratatui::layout::Rect {
    use ratatui::layout::{Constraint, Direction, Layout};

    let popup_layout = Layout::default()
        .direction(Direction::Vertical)
        .constraints([
            Constraint::Percentage((100 - percent_y) / 2),
            Constraint::Percentage(percent_y),
            Constraint::Percentage((100 - percent_y) / 2),
        ])
        .split(area);

    Layout::default()
        .direction(Direction::Horizontal)
        .constraints([
            Constraint::Percentage((100 - percent_x) / 2),
            Constraint::Percentage(percent_x),
            Constraint::Percentage((100 - percent_x) / 2),
        ])
        .split(popup_layout[1])[1]
}
```
