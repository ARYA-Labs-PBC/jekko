Absolutely — this changes my recommendation in a big way.

My upgraded recommendation is: **keep `TUI_BACKEND_API.md` as the contract, but build the visual/client layer as a new Rust-native Ratatui frontend.** The current extracted code is a SolidJS/OpenTUI app, so Ratatui should not be treated as a small styling patch. To leverage Ratatui properly, it should become the **new renderer/app shell**.

Ratatui is a strong fit because it is an immediate-mode Rust TUI library, built around `Terminal::draw`, `Frame`, widgets, layout, buffers, and backend abstractions. Its docs describe widgets as composable/nestable UI building blocks, with built-ins such as `Block`, `Paragraph`, `List`, `Table`, `Tabs`, `Scrollbar`, `Gauge`, `Canvas`, `Chart`, and more. It also supports terminal backends including Crossterm, Termion, Termwiz, and a `TestBackend` for UI testing. ([Ratatui][1])

Ratatui also gives you the kind of architecture you want for a premium coding app: immediate-mode rendering, efficient double-buffered screen flushing, custom widgets, stateful widgets, terminal input, mouse capture, alternate screen support, and snapshot-testable UI surfaces. ([Ratatui][2])

# 6. New Rust App Loop

## `src/main.rs`

```rust
use color_eyre::Result;
use crossterm::{
    event::{self, Event},
    execute,
    terminal::{
        disable_raw_mode, enable_raw_mode, EnterAlternateScreen, LeaveAlternateScreen,
    },
};
use ratatui::{
    backend::CrosstermBackend,
    Terminal,
};
use std::{io, time::Duration};

mod app;
mod action;
mod backend;
mod layout;
mod theme;
mod widgets;

use app::App;
use action::Action;

#[tokio::main]
async fn main() -> Result<()> {
    color_eyre::install()?;

    enable_raw_mode()?;

    let mut stdout = io::stdout();
    execute!(stdout, EnterAlternateScreen, event::EnableMouseCapture)?;

    let backend = CrosstermBackend::new(stdout);
    let mut terminal = Terminal::new(backend)?;

    let mut app = App::new().await?;

    let result = run(&mut terminal, &mut app).await;

    disable_raw_mode()?;
    execute!(
        terminal.backend_mut(),
        LeaveAlternateScreen,
        event::DisableMouseCapture
    )?;
    terminal.show_cursor()?;

    result
}

async fn run<B: ratatui::backend::Backend>(
    terminal: &mut Terminal<B>,
    app: &mut App,
) -> Result<()> {
    loop {
        terminal.draw(|frame| {
            widgets::root::render(frame, app);
        })?;

        if app.should_quit {
            break;
        }

        if event::poll(Duration::from_millis(16))? {
            match event::read()? {
                Event::Key(key) => {
                    let action = app.keymap.map_key(key);
                    app.dispatch(action).await?;
                }
                Event::Mouse(mouse) => {
                    app.dispatch(Action::Mouse(mouse)).await?;
                }
                Event::Resize(width, height) => {
                    app.dispatch(Action::Resize { width, height }).await?;
                }
                _ => {}
            }
        }

        while let Ok(event) = app.backend_events.try_recv() {
            app.apply_backend_event(event);
        }
    }

    Ok(())
}
```

This gives you a clean, deterministic app loop: render, read input, dispatch actions, apply backend events.

# 7. Core App State

## `src/app.rs`

```rust
use color_eyre::Result;
use tokio::sync::mpsc;

use crate::{
    action::Action,
    backend::{BackendClient, BackendEvent},
    keymap::Keymap,
    theme::{ThemeMode, ThemePack},
};

pub struct App {
    pub should_quit: bool,
    pub route: Route,
    pub theme: ThemePack,
    pub theme_mode: ThemeMode,
    pub keymap: Keymap,

    pub session_id: Option<String>,
    pub messages: Vec<MessageViewModel>,
    pub rail: ActivityRailState,
    pub composer: ComposerState,
    pub dialogs: DialogStack,
    pub toasts: ToastStack,

    pub client: BackendClient,
    pub backend_events: mpsc::Receiver<BackendEvent>,
}

impl App {
    pub async fn new() -> Result<Self> {
        let client = BackendClient::connect().await?;
        let backend_events = client.subscribe_events().await?;

        Ok(Self {
            should_quit: false,
            route: Route::Home,
            theme: ThemePack::lumen(),
            theme_mode: ThemeMode::Dark,
            keymap: Keymap::default(),

            session_id: None,
            messages: Vec::new(),
            rail: ActivityRailState::default(),
            composer: ComposerState::default(),
            dialogs: DialogStack::default(),
            toasts: ToastStack::default(),

            client,
            backend_events,
        })
    }

    pub async fn dispatch(&mut self, action: Action) -> Result<()> {
        match action {
            Action::Quit => self.should_quit = true,

            Action::ToggleThemeMode => {
                self.theme_mode = self.theme_mode.toggle();
                self.toasts.info(match self.theme_mode {
                    ThemeMode::Dark => "Dark mode enabled",
                    ThemeMode::Light => "White mode enabled",
                });
            }

            Action::ToggleRail => {
                self.rail.visible = !self.rail.visible;
            }

            Action::SubmitPrompt => {
                let Some(session_id) = self.session_id.clone() else {
                    return Ok(());
                };

                let text = self.composer.take_text();
                if !text.trim().is_empty() {
                    self.client.submit_prompt(&session_id, text).await?;
                }
            }

            Action::OpenCommandPalette => {
                self.dialogs.open(Dialog::CommandPalette);
            }

            Action::OpenThemePicker => {
                self.dialogs.open(Dialog::ThemePicker);
            }

            Action::BackendRefresh => {
                self.refresh().await?;
            }

            Action::Resize { width, height } => {
                self.rail.apply_responsive_size(width, height);
            }

            Action::Mouse(mouse) => {
                self.handle_mouse(mouse);
            }

            Action::None => {}
        }

        Ok(())
    }

    fn apply_backend_event(&mut self, event: BackendEvent) {
        match event {
            BackendEvent::SessionMessageDelta(delta) => {
                self.messages.apply_delta(delta);
            }
            BackendEvent::PermissionCreated(permission) => {
                self.dialogs.open(Dialog::Permission(permission));
            }
            BackendEvent::QuestionCreated(question) => {
                self.dialogs.open(Dialog::Question(question));
            }
            BackendEvent::ConfigUpdated(config) => {
                self.apply_config(config);
            }
            _ => {}
        }
    }
}
```

## `src/widgets/root.rs`

```rust
use ratatui::{
    layout::{Constraint, Direction, Layout, Rect},
    Frame,
};

use crate::{
    app::App,
    widgets::{
        activity_rail, command_bar, composer, dialog, session_canvas, status_bar, toast,
    },
};

pub fn render(frame: &mut Frame, app: &mut App) {
    let area = frame.area();
    let theme = app.theme.get(app.theme_mode);

    let vertical = Layout::default()
        .direction(Direction::Vertical)
        .constraints([
            Constraint::Length(3), // command bar
            Constraint::Min(0),    // main body
            Constraint::Length(app.composer.height()),
            Constraint::Length(1), // status bar
        ])
        .split(area);

    command_bar::render(frame, app, vertical[0]);

    let rail_width = app.rail.width_for(area.width);

    let main = if app.rail.visible && area.width >= 100 {
        let horizontal = Layout::default()
            .direction(Direction::Horizontal)
            .constraints([
                Constraint::Length(rail_width),
                Constraint::Min(40),
            ])
            .split(vertical[1]);

        activity_rail::render(frame, app, horizontal[0]);
        horizontal[1]
    } else {
        vertical[1]
    };

    session_canvas::render(frame, app, main);
    composer::render(frame, app, vertical[2]);
    status_bar::render(frame, app, vertical[3]);

    toast::render(frame, app, area);

    if app.rail.visible && area.width < 100 {
        render_rail_overlay(frame, app, vertical[1]);
    }

    dialog::render_stack(frame, app, area);
}

fn render_rail_overlay(frame: &mut Frame, app: &mut App, area: Rect) {
    let rail_width = app.rail.width_for(area.width).min(area.width);
    let rail = Rect {
        x: area.x,
        y: area.y,
        width: rail_width,
        height: area.height,
    };

    activity_rail::render_overlay(frame, app, rail);
}
```

# 15. Message Timeline

## `src/widgets/message_timeline.rs`

```rust
use ratatui::{
    layout::{Constraint, Direction, Layout, Rect},
    style::{Modifier, Style},
    text::{Line, Span, Text},
    widgets::{Block, Borders, Paragraph, Wrap},
    Frame,
};

use crate::{app::App, backend::models::MessageRole};

pub fn render(frame: &mut Frame, app: &App, area: Rect) {
    let theme = app.theme.get(app.theme_mode);

    let mut cursor_y = area.y;

    for message in app.visible_messages() {
        let remaining = area.bottom().saturating_sub(cursor_y);
        if remaining == 0 {
            break;
        }

        let height = estimate_height(&message.content, area.width).min(remaining);
        let rect = Rect {
            x: area.x + 2,
            y: cursor_y,
            width: area.width.saturating_sub(4),
            height,
        };

        match message.role {
            MessageRole::User => render_user(frame, app, rect, &message.content),
            MessageRole::Assistant => render_assistant(frame, app, rect, &message.content),
            MessageRole::Tool => render_tool(frame, app, rect, message),
        }

        cursor_y = cursor_y.saturating_add(height + 1);
    }
}

fn render_user(frame: &mut Frame, app: &App, area: Rect, content: &str) {
    let theme = app.theme.get(app.theme_mode);

    let block = Block::default()

        .style(Style::default().bg(theme.surface));

        Line::from(vec![
            Span::styled("◆ You", Style::default().fg(theme.primary).add_modifier(Modifier::BOLD)),
        ]),
        Line::raw(""),
        Line::from(content.to_string()),
    ]);

    let paragraph = Paragraph::new(text)
        .style(Style::default().fg(theme.text).bg(theme.surface))
```

# 17. Diff Viewer

Use `similar` to parse hunks and render lines with Ratatui `Span`s. Keep split/unified view behavior.

## `src/widgets/diff_view.rs`

```rust
use ratatui::{
    layout::Rect,
    style::Style,
    text::{Line, Span, Text},
    widgets::{Block, Borders, Paragraph, Wrap},
    Frame,
};

use crate::app::App;

pub fn render_unified(frame: &mut Frame, app: &App, area: Rect, file: &str, diff: &str) {
    let theme = app.theme.get(app.theme_mode);

    let block = Block::default()
        .borders(Borders::ALL)
        .title(format!("← Edit {}", file))
        .border_style(Style::default().fg(theme.accent))
        .style(Style::default().bg(theme.panel));

    let inner = block.inner(area);
    frame.render_widget(block, area);

    let lines = diff.lines().map(|line| {
        if line.starts_with('+') && !line.starts_with("+++") {
            Line::from(Span::styled(
                line.to_string(),
                Style::default().fg(theme.success).bg(theme.diff_added_bg),
            ))
        } else if line.starts_with('-') && !line.starts_with("---") {
            Line::from(Span::styled(
                line.to_string(),
                Style::default().fg(theme.error).bg(theme.diff_removed_bg),
            ))
        } else if line.starts_with("@@") {
            Line::from(Span::styled(
                line.to_string(),
                Style::default().fg(theme.secondary).bg(theme.diff_context_bg),
            ))
        } else {
            Line::from(Span::styled(
                line.to_string(),
                Style::default().fg(theme.muted).bg(theme.diff_context_bg),
            ))
        }
    });

    let paragraph = Paragraph::new(Text::from_iter(lines))
        .wrap(Wrap { trim: false })
        .style(Style::default().bg(theme.panel));

    frame.render_widget(paragraph, inner);
}
```

# 18. Dialogs and Command Palette

Ratatui’s rendering model supports popups by drawing a `Clear` widget over a rectangular region and then drawing a modal on top. The docs specifically mention `Clear` as a built-in widget useful for rendering over previously drawn widgets. ([Ratatui][1])

## `src/widgets/dialog.rs`

```rust
use ratatui::{
    layout::{Alignment, Constraint, Direction, Layout, Rect},
    style::{Modifier, Style},
    widgets::{Block, Borders, Clear, Paragraph},
    Frame,
};

use crate::app::App;

pub fn render_stack(frame: &mut Frame, app: &mut App, area: Rect) {
    let Some(dialog) = app.dialogs.top() else {
        return;
    };

    let popup = centered_rect(72, 70, area);
    frame.render_widget(Clear, popup);

    match dialog {
        Dialog::CommandPalette => crate::widgets::command_palette::render(frame, app, popup),
        Dialog::ThemePicker => crate::widgets::theme_picker::render(frame, app, popup),
        Dialog::Permission(permission) => {
            crate::widgets::permission_card::render(frame, app, popup, permission)
        }
        Dialog::Question(question) => {
            crate::widgets::question_card::render(frame, app, popup, question)
        }
    }
}

fn centered_rect(percent_x: u16, percent_y: u16, area: Rect) -> Rect {
    let vertical = Layout::default()
        .direction(Direction::Vertical)
        .constraints([
            Constraint::Percentage((100 - percent_y) / 2),
            Constraint::Percentage(percent_y),
            Constraint::Percentage((100 - percent_y) / 2),
        ])
        .split(area);

    let horizontal = Layout::default()
        .direction(Direction::Horizontal)
        .constraints([
            Constraint::Percentage((100 - percent_x) / 2),
            Constraint::Percentage(percent_x),
            Constraint::Percentage((100 - percent_x) / 2),
        ])
        .split(vertical[1]);

    horizontal[1]
}
```

Keep the current TypeScript backend worker and make the Rust TUI launch it.

Goal:

```txt
lumen tui
  → starts existing backend worker
  → worker exposes HTTP/SSE
  → Rust Ratatui app connects to worker
```

This is the shortest path to Ratatui.

## Path B: Best long-term

Extract the backend into a standalone daemon/server.

Add:

```txt
crates/lumen-daemon/
crates/lumen-tui/
```

The CLI becomes:

```txt
lumen daemon start
lumen tui
```

or:

```txt
lumen
```

which launches both.

This is cleaner and much more “real product” than a Rust UI shell that secretly depends on the old TS frontend internals.

This gives you the best chance of building something that feels like **a completely new Rust-native coding environment**, not a fork with a new paint job.

The earlier file-change report is still useful for an incremental path, but the **best path** is now:

```txt
new crates/lumen-tui Ratatui frontend
existing TUI_BACKEND_API.md backend contract
old OpenTUI renderer deprecated or kept only as temporary fallback
```

That is the direction I’d choose for the world’s greatest Rust TUI coding app.
