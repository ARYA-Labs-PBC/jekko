## Core recommendation

You’re right to dislike the current setup. The code is still built around **user-selected provider/model state**, while the product you want is **key-file-first, auto-routed, free-safe, paid-opt-in**.

The new default should be:

```text
User pastes keys → Jekko detects providers → Jekko routes automatically → user never picks a model
```

Normal users should only touch:

```text
~/.config/jekko/keys.env
~/.config/jekko/model-policy.jsonc
```

`keys.env` is the central secret file:

```dotenv
OPENAI_API_KEY=
ANTHROPIC_API_KEY=
GOOGLE_GENERATIVE_AI_API_KEY=
JEKKO_API_KEY=

# Developer/protected router entitlement
JEKKO_DEVELOPER_KEY=
```

`model-policy.jsonc` is the non-secret paid-model opt-in:

```jsonc
{
  "$schema": "https://jekko.ai/model-policy.json",

  // Paid models are never used unless listed here.
  "paid_models": [],

  // Optional hard deny list.
  "never_models": [],

  "routing": {
    "mode": "auto",
    "optimize": "cost",
    "prefer_free": true
  }
}
```

Free/public models should be eligible automatically. Paid models should be used only when explicitly listed.

## Target architecture

Add a virtual default model:

```ts
{ providerID: "auto", modelID: "smart" }
```

The TUI and ACP should show this as:

```text
Auto
Jekko Smart
Auto · Free-first
Auto · Paid allowed
```

Then `session/llm.ts` should resolve `auto/smart` to a concrete provider/model immediately before the LLM call.

This is the best insertion point because `session/llm.ts` already receives the selected model, loads the provider SDK, applies provider transforms, and calls `streamText`.

## The critical data-model change

Do **not** only store the resolved concrete model in session history.

Right now, the system stores concrete model state and then `lastModel()` reuses it. A naive auto-router would work on the first turn, then become sticky on the second turn because the last resolved model becomes the next requested model.

You need separate fields:

```ts
requestedModel: {
  providerID: "auto",
  modelID: "smart"
}

resolvedModel: {
  providerID: "openai",
  modelID: "..."
}
```

or equivalent.

Without that split, auto-routing silently disables itself after one response.

## Free vs developer behavior

### Free/basic mode

Without a developer key:

```text
- Free/public Jekko models are eligible.
- One paid-key provider can be used.
- Extra keys may be present but are not used for multi-provider balancing.
- Paid models require model-policy allowlist.
- Best free capable model wins when available.
```

### Developer/protected mode

With a valid developer key:

```text
- Unlimited provider keys may participate.
- Full balancing/failover/health/quota logic comes from protected code.
- Public repo only builds candidates and calls protected router.
- Paid model allowlist still controls spending unless product policy says otherwise.
```

## Codex/Gemini/OpenAI/Claude handling

The current tree’s Codex support is not a simple `keys.env` API-key provider. It is implemented through the OpenAI auth/plugin path. So:

```text
OpenAI API key       → keys.env / OPENAI_API_KEY
Gemini API key       → keys.env / GOOGLE_GENERATIVE_AI_API_KEY
Claude API key       → keys.env / ANTHROPIC_API_KEY
Codex / ChatGPT auth → keep OAuth/plugin flow, advanced setup
```

Once Codex OAuth exists, the router can include it as a candidate source, but the system should not pretend a pasted API key “connects Codex.”

## Recommended implementation phases

1. **Cleanup:** delete `._*`, add secret redaction tests.
2. **Key file + model policy:** add `keys.env` parser, policy loader, `jekko keys status`.
3. **Basic router:** add `ModelRouter.Service`, `auto/smart`, free-first and one-provider basic routing.
4. **Session fix:** split requested vs resolved model so auto-routing does not become sticky.
5. **TUI simplification:** show `Auto`; replace provider/model dialogs with key/status UI.
6. **ACP/server:** expose Auto and route-status endpoints.
7. **Protected Jnoccio router:** load protected code or call Jnoccio Fusion RPC when developer key is valid.
8. **Migration/deprecation:** migrate API keys out of config/auth API-key storage, then hide or delete manual selection paths after compatibility period.

Bottom line: make `auto/smart` the only normal model, make `keys.env` the only normal secret setup, make `model-policy.jsonc` the paid safety valve, and keep the unlimited paid Jnoccio router behind protected code.
