# Jekko OpenTUI/Bun/Vite → Rust/Ratatui Migration Engineering Plan

## 1. Executive Summary

The uploaded archive is a focused “touchpoint” snapshot rather than a complete repository. It contains the current OpenTUI/Solid/Bun TUI implementation, Bun/Nix/CI build infrastructure, tests, scripts, and package metadata. It does **not** contain a Cargo workspace or root Rust application files, so the Rust crate layout below is a proposed target structure. If the full repository already has a Rust workspace, fold these crates/modules into that existing workspace instead of creating a second one.

The migration is not a small dependency swap. The current TUI is a TypeScript/Solid/OpenTUI retained/component-style UI that depends on Bun for runtime, build, test, plugin loading, and packaged binaries. Ratatui is a Rust immediate-mode TUI library: every frame is rendered from current application state, and Ratatui compares the current and previous buffers to write only terminal changes. That means the migration should replace the current JSX/provider/plugin-rendering tree with a Rust `App` state model, explicit layout functions, typed widgets, an event loop, and Rust tests. ([Docs.rs][1])

My recommended end state is:

```text
No OpenTUI
No @opentui/core
No @opentui/solid
No opentui-spinner
No Solid JSX for the TUI
No Bun runtime
No bun.lock
No bunfig.toml
No Bun-based CI actions
No Bun-based binary build
No Vite
Rust binary built by Cargo
Ratatui + Crossterm terminal UI
Rust-owned state, routing, keybinds, dialogs, prompt, session view, plugin slots, and tests
```

I found **zero `vite` / `Vite` references** in the uploaded snapshot, so Vite removal is mostly a guardrail item for the wider repo. OpenTUI and Bun removal are substantial: the snapshot has OpenTUI/Solid references across the TUI tree, package metadata, tests, build scripts, plugin APIs, Nix derivations, and CI.

### 2.3 Current architecture in plain terms

The current TUI is centered around:

```text
packages/jekko/src/cli/cmd/tui/app.tsx
packages/jekko/src/cli/cmd/tui/app-view.tsx
packages/jekko/src/cli/cmd/tui/routes/*

packages/jekko/src/cli/cmd/tui/context/*
packages/jekko/src/cli/cmd/tui/plugin/*
packages/jekko/src/cli/cmd/tui/feature-plugins/*
```

The current boot path creates an OpenTUI `CliRenderer`, enables mouse/kitty keyboard options, detects palette/theme mode, renders Solid JSX, mounts a large provider tree, initializes plugins, then routes among Home, Shell, Session, and plugin routes.

OpenTUI itself is a native terminal UI core written in Zig with TypeScript bindings; its monorepo includes `@opentui/core` and `@opentui/solid`, and the official quickstart is Bun-based. That matches the current Jekko implementation very closely. ([GitHub][2])

The plugin system is also OpenTUI/Solid-specific. OpenTUI’s Solid plugin docs describe runtime-loaded external plugins as a Bun drop-in that shares `@opentui/solid`, `@opentui/core`, `solid-js`, and related runtime modules. A strict “remove Bun 100%” goal means the current external TSX/Solid plugin model cannot remain as-is. ([OpenTUI][3])

## 3. Target Technical Direction

### 3.1 Use Ratatui + Crossterm as the canonical UI stack

Recommended Rust UI stack:

```toml
ratatui = "0.30"
crossterm = "0.29"
tokio = "1"
color-eyre = "0.6"
tracing = "0.1"
tracing-subscriber = "0.3"
serde = "1"
serde_json = "1"
serde_jsonc = "1"
anyhow = "1"
thiserror = "2"
ratatui-textarea = "0.9"
fuzzy-matcher = "0.3" or nucleo-matcher
arboard or copypasta
notify
portable-pty
reqwest
tokio-tungstenite
```

Ratatui uses Crossterm by default, and its backend model supports terminal drawing, cursor manipulation, size inspection, raw mode, alternate screen, mouse capture, and a `TestBackend` for UI tests. ([Docs.rs][1])

Use Ratatui’s immediate rendering model deliberately:

```text
State lives in Rust structs.
Events mutate state.
Every frame renders the whole visible UI from that state.
Ratatui diffs buffers and writes only terminal changes.
```

That is the correct conceptual replacement for Solid signals/providers. Ratatui’s `Terminal::draw` docs specifically require the render callback to fully render the entire frame, because buffer comparison depends on complete frame output. ([Docs.rs][4])

### 3.2 Do not try to emulate Solid in Rust

Avoid building a mini JSX framework. Ratatui works best when the UI is decomposed into:

```text
App state
Route state
Domain state
Widget render functions
Widget state structs
Explicit layout functions
Event handlers
Integration tests using TestBackend and PTY tests
```

The replacement should look like a canonical Rust TUI, not TypeScript JSX translated line-for-line.

### 3.3 Replace OpenTUI flex layout with explicit Ratatui rectangles

OpenTUI layout currently uses JSX props such as `flexGrow`, `flexDirection`, `paddingLeft`, `paddingRight`, `position="absolute"`, `zIndex`, and `scrollbox`.

Ratatui uses terminal coordinates and rectangular areas. The coordinate system starts at `(0, 0)` in the top-left corner, layouts split terminal areas using constraints, and widgets render into those rectangles. ([Ratatui][5])

Therefore, the migration should create a small set of layout helpers that preserve the current visual constants:

```text
Header height: 1 row
Footer band: usually 2–3 rows depending route
Horizontal padding: 2 columns
Left shell panel:
  width < 80    => hidden
  width < 100   => overlay width 28 when visible
  width < 120   => fixed width 28
  width < 160   => fixed width 38
  width >= 160  => fixed width 44
Dialog widths:
  medium => 60
  large  => 88
  xlarge => 116
```

### 3.4 Preserve functionality with a Rust state/store model

The current provider tree in `app.tsx` should become explicit Rust state modules:

```text
ArgsProvider              -> app::Args
ExitProvider              -> app::ExitState / shutdown flow
KVProvider                -> state::kv
ToastProvider             -> widgets::toast::ToastState
RouteProvider             -> router::Router
TuiConfigProvider         -> config::TuiConfig
SDKProvider               -> sdk::Client / sdk::EventSource
ProjectProvider           -> state::project
SyncProvider / SyncV2     -> state::sync
ThemeProvider             -> theme::ThemeStore
LocalProvider             -> state::local
KeybindProvider           -> keybind::Keymap
PromptStashProvider       -> prompt::stash
DialogProvider            -> widgets::dialog::DialogStack
CommandProvider           -> command::CommandRegistry
FrecencyProvider          -> prompt::frecency
PromptHistoryProvider     -> prompt::history
PromptRefProvider         -> prompt::PromptRef equivalent
EditorContextProvider     -> editor::EditorState
```

A good internal Rust slot model:

```rust
use ratatui::{Frame, layout::Rect};

#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
pub enum SlotName {
    App,
    HomeLogo,
    HomeBottom,
    HomeFooter,
    SidebarTitle,
    SidebarContent,
    SidebarFooter,
    ShellLeftTabs,
    ShellLeftActivePane,
    ShellCenterFeed,
    ShellFooter,
    SessionPrompt,
    SessionPromptRight,
}

#[derive(Debug, Clone, Copy)]
pub enum SlotMode {
    Additive,
    Replace,
    SingleWinner,
}

pub struct SlotProps<'a> {
    pub session_id: Option<&'a str>,
    pub active_pane: Option<&'a str>,
    pub left_width: u16,
    pub left_content_width: u16,
    pub center_width: u16,
    pub center_content_width: u16,
}

pub trait SlotRenderer: Send + Sync {
    fn id(&self) -> &'static str;
    fn slot(&self) -> SlotName;
    fn mode(&self) -> SlotMode {
        SlotMode::Additive
    }
    fn render(&self, frame: &mut Frame, area: Rect, app: &crate::app::App, props: SlotProps<'_>);
}

pub struct SlotRegistry {
    renderers: Vec<Box<dyn SlotRenderer>>,
}

impl SlotRegistry {
    pub fn new() -> Self {
        Self { renderers: Vec::new() }
    }

    pub fn register<R>(&mut self, renderer: R)
    where
        R: SlotRenderer + 'static,
    {
        self.renderers.push(Box::new(renderer));
    }

    pub fn render(
        &self,
        slot: SlotName,
        frame: &mut Frame,
        area: Rect,
        app: &crate::app::App,
        props: SlotProps<'_>,
    ) {
        let matches = self.renderers.iter().filter(|r| r.slot() == slot);

        let mut single_winner: Option<&Box<dyn SlotRenderer>> = None;
        for renderer in matches {
            match renderer.mode() {
                SlotMode::Replace | SlotMode::SingleWinner => {
                    single_winner = Some(renderer);
                }
                SlotMode::Additive => {
                    renderer.render(frame, area, app, props);
                }
            }
        }

        if let Some(renderer) = single_winner {
            renderer.render(frame, area, app, props);
        }
    }
}
```

```rust
// crates/jekko-tui/src/terminal.rs

use std::io::{self, Stdout};

use color_eyre::eyre::Result;
use crossterm::{
    cursor::{Hide, Show},
    event::{DisableMouseCapture, EnableMouseCapture},
    execute,
    terminal::{
        disable_raw_mode, enable_raw_mode, EnterAlternateScreen, LeaveAlternateScreen,
    },
};
use ratatui::{backend::CrosstermBackend, Terminal};

pub type Backend = CrosstermBackend<Stdout>;
pub type TuiTerminal = Terminal<Backend>;

pub struct TerminalGuard {
    terminal: TuiTerminal,
    mouse_enabled: bool,
}

impl TerminalGuard {
    pub fn enter(mouse_enabled: bool) -> Result<Self> {
        color_eyre::install()?;
        enable_raw_mode()?;

        let mut stdout = io::stdout();

        if mouse_enabled {
            execute!(stdout, EnterAlternateScreen, EnableMouseCapture, Hide)?;
        } else {
            execute!(stdout, EnterAlternateScreen, Hide)?;
        }

        let backend = CrosstermBackend::new(stdout);
        let mut terminal = Terminal::new(backend)?;
        terminal.clear()?;

        Ok(Self {
            terminal,
            mouse_enabled,
        })
    }

    pub fn terminal_mut(&mut self) -> &mut TuiTerminal {
        &mut self.terminal
    }
}

impl Drop for TerminalGuard {
    fn drop(&mut self) {
        let _ = disable_raw_mode();

        let backend = self.terminal.backend_mut();

        if self.mouse_enabled {
            let _ = execute!(backend, Show, DisableMouseCapture, LeaveAlternateScreen);
        } else {
            let _ = execute!(backend, Show, LeaveAlternateScreen);
        }

        let _ = self.terminal.show_cursor();
    }
}
```

Use manual setup rather than `ratatui::run()` because Jekko needs fine-grained control over mouse capture, title restore, startup logging, fatal restore, and PTY test behavior. Ratatui’s docs support both convenience initialization and manual construction for applications needing finer control. ([Docs.rs][4])

## 7.2 App state and event loop

```rust
// crates/jekko-tui/src/app.rs

use std::time::{Duration, Instant};

use color_eyre::eyre::Result;
use crossterm::event::{self, Event, KeyCode, KeyEvent, KeyEventKind, KeyModifiers};
use ratatui::{Frame, layout::Rect, widgets::Clear};

use crate::{
    keybind::Keymap,
    router::{Route, Router},
    terminal::TuiTerminal,
    theme::ThemeStore,
    widgets::{dialog::DialogStack, toast::ToastState},
};

pub struct App {
    pub router: Router,
    pub keymap: Keymap,
    pub theme: ThemeStore,
    pub dialogs: DialogStack,
    pub toasts: ToastState,
    pub shell: crate::routes::shell::ShellState,
    pub session: crate::routes::session::SessionState,
    pub home: crate::routes::home::HomeState,
    pub should_quit: bool,
    first_frame_logged: bool,
    started_at: Instant,
}

impl App {
    pub fn new(args: crate::state::args::Args) -> Self {
        Self {
            router: Router::new(args.initial_route()),
            keymap: Keymap::default(),
            theme: ThemeStore::load_or_default(),
            dialogs: DialogStack::default(),
            toasts: ToastState::default(),
            shell: crate::routes::shell::ShellState::default(),
            session: crate::routes::session::SessionState::default(),
            home: crate::routes::home::HomeState::default(),
            should_quit: false,
            first_frame_logged: false,
            started_at: Instant::now(),
        }
    }

    pub fn run(&mut self, terminal: &mut TuiTerminal) -> Result<()> {
        let tick_rate = Duration::from_millis(16); // ~60 FPS upper bound
        let mut last_tick = Instant::now();

        while !self.should_quit {
            terminal.draw(|frame| self.render(frame))?;

            if !self.first_frame_logged {
                self.first_frame_logged = true;
                tracing::info!(
                    elapsed_ms = self.started_at.elapsed().as_millis(),
                    "tui first frame"
                );
            }

            let timeout = tick_rate
                .checked_sub(last_tick.elapsed())
                .unwrap_or_else(|| Duration::from_millis(0));

            if event::poll(timeout)? {
                let event = event::read()?;
                self.handle_event(event)?;
            }

            if last_tick.elapsed() >= tick_rate {
                self.tick();
                last_tick = Instant::now();
            }
        }

        Ok(())
    }

    fn tick(&mut self) {
        self.toasts.retain_live();
        self.session.tick();
        self.shell.tick();
    }

    fn handle_event(&mut self, event: Event) -> Result<()> {
        match event {
            Event::Key(key) if key.kind == KeyEventKind::Press => self.handle_key(key),
            Event::Mouse(mouse) => self.handle_mouse(mouse),
            Event::Resize(width, height) => {
                tracing::debug!(width, height, "terminal resized");
                Ok(())
            }
            _ => Ok(()),
        }
    }

    fn handle_key(&mut self, key: KeyEvent) -> Result<()> {
        if self.dialogs.is_open() {
            if self.dialogs.handle_key(key, &mut self.toasts) {
                return Ok(());
            }
        }

        if key.modifiers.contains(KeyModifiers::CONTROL) && key.code == KeyCode::Char('c') {
            self.should_quit = true;
            return Ok(());
        }

        if self.keymap.matches("command.open", key) {
            self.dialogs.open_command_palette();
            return Ok(());
        }

        if self.keymap.matches("route.back", key) {
            self.router.navigate_back();
            return Ok(());
        }

        if self.keymap.matches("shell.left.toggle", key) {
            self.shell.left_visible = !self.shell.left_visible;
            return Ok(());
        }

        match self.router.current() {
            Route::Home => self.home.handle_key(key, &mut self.router),
            Route::Shell => self.shell.handle_key(key, &mut self.router),
            Route::Session { .. } => self.session.handle_key(key, &mut self.router),
            Route::Plugin { .. } => Ok(()),
        }
    }

    fn handle_mouse(&mut self, mouse: crossterm::event::MouseEvent) -> Result<()> {
        if self.dialogs.handle_mouse(mouse) {
            return Ok(());
        }
        Ok(())
    }

    fn render(&mut self, frame: &mut Frame) {
        let area = frame.area();
        frame.render_widget(Clear, area);

        crate::routes::render_root(frame, area, self);

        if self.dialogs.is_open() {
            self.dialogs.render(frame, area, &self.theme.current);
        }

        self.toasts.render(frame, area, &self.theme.current);
    }
}
```

```rust
// crates/jekko-tui/src/theme.rs

use serde::Deserialize;
use ratatui::style::{Color, Style};

#[derive(Debug, Clone, Deserialize)]
pub struct ThemeJson {
    pub primary: String,
    pub secondary: String,
    pub accent: String,
    pub error: String,
    pub warning: String,
    pub success: String,
    pub info: String,
    pub text: String,
    #[serde(rename = "textMuted")]
    pub text_muted: String,
    pub background: String,
    #[serde(rename = "backgroundPanel")]
    pub background_panel: String,
    #[serde(rename = "backgroundElement")]
    pub background_element: String,
    #[serde(rename = "backgroundMenu")]
    pub background_menu: String,
    pub border: String,
    #[serde(rename = "borderActive")]
    pub border_active: String,
    #[serde(rename = "borderSubtle")]
    pub border_subtle: String,
}

#[derive(Debug, Clone)]
pub struct Theme {
    pub primary: Color,
    pub secondary: Color,
    pub accent: Color,
    pub error: Color,
    pub warning: Color,
    pub success: Color,
    pub info: Color,
    pub text: Color,
    pub text_muted: Color,
    pub background: Color,
    pub background_panel: Color,
    pub background_element: Color,
    pub background_menu: Color,
    pub border: Color,
    pub border_active: Color,
    pub border_subtle: Color,
}

impl Theme {
    pub fn from_json(input: ThemeJson) -> anyhow::Result<Self> {
        Ok(Self {
            primary: parse_hex(&input.primary)?,
            secondary: parse_hex(&input.secondary)?,
            accent: parse_hex(&input.accent)?,
            error: parse_hex(&input.error)?,
            warning: parse_hex(&input.warning)?,
            success: parse_hex(&input.success)?,
            info: parse_hex(&input.info)?,
            text: parse_hex(&input.text)?,
            text_muted: parse_hex(&input.text_muted)?,
            background: parse_hex(&input.background)?,
            background_panel: parse_hex(&input.background_panel)?,
            background_element: parse_hex(&input.background_element)?,
            background_menu: parse_hex(&input.background_menu)?,
            border: parse_hex(&input.border)?,
            border_active: parse_hex(&input.border_active)?,
            border_subtle: parse_hex(&input.border_subtle)?,
        })
    }

    pub fn root_style(&self) -> Style {
        Style::default().fg(self.text).bg(self.background)
    }
}

fn parse_hex(input: &str) -> anyhow::Result<Color> {
    let value = input.trim().trim_start_matches('#');

    if value.len() != 6 {
        anyhow::bail!("expected #RRGGBB color, got {input}");
    }

    let r = u8::from_str_radix(&value[0..2], 16)?;
    let g = u8::from_str_radix(&value[2..4], 16)?;
    let b = u8::from_str_radix(&value[4..6], 16)?;

    Ok(Color::Rgb(r, g, b))
}

#[derive(Debug, Clone)]
pub struct ThemeStore {
    pub current: Theme,
    pub selected: String,
    pub mode: ThemeMode,
}

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum ThemeMode {
    Dark,
    Light,
}

impl ThemeStore {
    pub fn load_or_default() -> Self {
        let json: ThemeJson = serde_json::from_str(include_str!("../assets/themes/jekko.json"))
            .expect("bundled default theme must parse");
        Self {
            current: Theme::from_json(json).expect("bundled default theme must be valid"),
            selected: "jekko".to_string(),
            mode: ThemeMode::Dark,
        }
    }
}
```

## 7.6 Shell layout: preserve current left-panel behavior

Current `shell-view.tsx` has this logic:

```text

width < 100   => overlay width 28 if visible
width < 120   => shown width 28
width < 160   => shown width 38
else          => shown width 44
```

Ratatui equivalent:

```rust
// crates/jekko-tui/src/routes/shell.rs

use ratatui::{
    Frame,
    layout::{Constraint, Layout, Rect},
    style::Style,
    widgets::{Block, Clear, Paragraph},
};

use crate::{app::App, widgets::footer_band::FooterBand};

#[derive(Debug, Clone, Copy)]
pub enum LeftPanel {
    Hidden,
    Shown { width: u16, overlay: bool },
}

#[derive(Debug, Default)]
pub struct ShellState {
    pub left_visible: bool,
    pub active_pane: ShellPane,
    pub jnoccio_available: bool,
}

#[derive(Debug, Default, Clone, Copy, PartialEq, Eq)]
pub enum ShellPane {
    #[default]
    Context,
    Zyal,
    Jankurai,
    Capability,
    History,
}

pub fn resolve_left(width: u16, visible: bool) -> LeftPanel {
    if width < 80 {
        LeftPanel::Hidden
    } else if width < 100 {
        if visible {
            LeftPanel::Shown {
                width: 28,
                overlay: true,
            }
        } else {
            LeftPanel::Hidden
        }
    } else if width < 120 {
        LeftPanel::Shown {
            width: 28,
            overlay: false,
        }
    } else if width < 160 {
        LeftPanel::Shown {
            width: 38,
            overlay: false,
        }
    } else {
        LeftPanel::Shown {
            width: 44,
            overlay: false,
        }
    }
}

pub fn render(frame: &mut Frame, area: Rect, app: &mut App) {
    let theme = &app.theme.current;

    let [main, footer] = Layout::vertical([
        Constraint::Min(0),
        Constraint::Length(3),
    ])
    .areas(area);

    let left = resolve_left(main.width, app.shell.left_visible);

    match left {
        LeftPanel::Hidden => {
            render_center(frame, main, app, 0);
        }
        LeftPanel::Shown { width, overlay: false } => {
            let [left_area, center_area] = Layout::horizontal([
                Constraint::Length(width),
                Constraint::Min(0),
            ])
            .areas(main);

            render_left_panel(frame, left_area, app, false);
            render_center(frame, center_area, app, width);
        }
        LeftPanel::Shown { width, overlay: true } => {
            render_center(frame, main, app, 0);

            let overlay_area = Rect {
                x: main.x,
                y: main.y,
                width: width.min(main.width),
                height: main.height,
            };

            frame.render_widget(Clear, overlay_area);
            render_left_panel(frame, overlay_area, app, true);
        }
    }

    frame.render_widget(
        FooterBand::new(theme.background_panel, theme.border_subtle),
        footer,
    );

    render_footer_hints(frame, footer, app);
}

fn render_left_panel(frame: &mut Frame, area: Rect, app: &App, overlay: bool) {
    let theme = &app.theme.current;

    frame.render_widget(
        Block::default().style(Style::default().bg(theme.background_panel)),
        area,
    );

    let inner = area.inner(ratatui::layout::Margin {
        horizontal: 2,
        vertical: 1,
    });

    let [tabs, body] = Layout::vertical([
        Constraint::Length(1),
        Constraint::Min(0),
    ])
    .areas(inner);

    crate::feature_plugins::shell::render_tabs(frame, tabs, app);
    crate::feature_plugins::shell::render_active_pane(frame, body, app, overlay);
}

fn render_center(frame: &mut Frame, area: Rect, app: &mut App, left_width: u16) {
    let theme = &app.theme.current;

    let inner = area.inner(ratatui::layout::Margin {
        horizontal: 2,
        vertical: 0,
    });

    if app.session.has_active_feed() {
        crate::feature_plugins::shell::render_activity_feed(frame, inner, app, left_width);
    } else {
        frame.render_widget(
            Paragraph::new("No active feed")
                .style(Style::default().fg(theme.text_muted).bg(theme.background)),
            center_line(inner),
        );
    }
}

fn render_footer_hints(frame: &mut Frame, area: Rect, app: &App) {
    use ratatui::{text::{Line, Span}, widgets::Paragraph};

    let theme = &app.theme.current;
    let inner = area.inner(ratatui::layout::Margin {
        horizontal: 2,
        vertical: 1,
    });

    let spans = vec![
        key(" Enter ", theme), Span::raw(" send  "),
        key(" Ctrl+P ", theme), Span::raw(" commands  "),
        key(" Tab ", theme), Span::raw(" pane  "),
        key(" Ctrl+B ", theme), Span::raw(" left  "),
        key(" Ctrl+Shift+T ", theme), Span::raw(" theme  "),
        key(" ? ", theme), Span::raw(" help  "),
        key(" Ctrl+C ", theme), Span::raw(" quit"),
    ];
```

## 7.8 Dialog overlay

Current `ui/dialog.tsx` uses:

```text
absolute overlay
zIndex 3000
translucent black background
centered panel
width medium 60 / large 88 / xlarge 116
Escape or Ctrl-C closes
focus restore
click outside closes unless selection is active
```

Ratatui cannot do real alpha compositing like `RGBA(0,0,0,150)`. The practical pixel-close approximation is `Clear` plus a full-screen dark style, then a centered panel.

```rust
// crates/jekko-tui/src/widgets/dialog.rs

use crossterm::event::{KeyCode, KeyEvent, KeyModifiers, MouseButton, MouseEvent, MouseEventKind};
use ratatui::{
    Frame,
    layout::{Constraint, Flex, Layout, Rect},
    style::Style,
    widgets::{Block, Borders, Clear, Paragraph},
};

use crate::{theme::Theme, widgets::toast::ToastState};

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum DialogSize {
    Medium,
    Large,
    XLarge,
}

impl DialogSize {
    pub fn width(self) -> u16 {
        match self {
            Self::Medium => 60,
            Self::Large => 88,
            Self::XLarge => 116,
        }
    }
}

#[derive(Debug)]
pub enum DialogKind {
    CommandPalette,
    Help,
    Alert { title: String, message: String },
    Confirm { title: String, message: String },
    Prompt { title: String, value: String },
    Select,
}

#[derive(Debug)]
pub struct DialogEntry {
    pub size: DialogSize,
    pub kind: DialogKind,
}

#[derive(Debug, Default)]
pub struct DialogStack {
    stack: Vec<DialogEntry>,
}

impl DialogStack {
    pub fn is_open(&self) -> bool {
        !self.stack.is_empty()
    }

    pub fn clear(&mut self) {
        self.stack.clear();
    }

    pub fn replace(&mut self, entry: DialogEntry) {
        self.stack.clear();
        self.stack.push(entry);
    }

    pub fn open_command_palette(&mut self) {
        self.replace(DialogEntry {
            size: DialogSize::Large,
            kind: DialogKind::CommandPalette,
        });
    }

    pub fn handle_key(&mut self, key: KeyEvent, _toasts: &mut ToastState) -> bool {
        if self.stack.is_empty() {
            return false;
        }

        let close =
            key.code == KeyCode::Esc ||
            (key.modifiers.contains(KeyModifiers::CONTROL) && key.code == KeyCode::Char('c'));

        if close {
            self.clear();
            return true;
        }

        true
    }

    pub fn handle_mouse(&mut self, mouse: MouseEvent) -> bool {
        if self.stack.is_empty() {
            return false;
        }

        if matches!(mouse.kind, MouseEventKind::Up(MouseButton::Left)) {
            // The render path records panel rect in a production version.
            // If click is outside panel, clear. Keep as stateful hit-test.
            return true;
        }

        true
    }

    pub fn render(&self, frame: &mut Frame, area: Rect, theme: &Theme) {
        let Some(entry) = self.stack.last() else {
            return;
        };

        frame.render_widget(Clear, area);
        frame.render_widget(
            Block::default().style(Style::default().bg(theme.background)),
            area,
        );

        let panel = centered_rect(area, entry.size.width(), panel_height(entry, area));

        frame.render_widget(Clear, panel);
        frame.render_widget(
            Block::default()
                .borders(Borders::ALL)
                .border_style(Style::default().fg(theme.border_active))
                .style(Style::default().bg(theme.background_panel)),
            panel,
        );

        let inner = panel.inner(ratatui::layout::Margin {
            horizontal: 2,
            vertical: 1,
        });

        match &entry.kind {
            DialogKind::Alert { title, message } => {
                frame.render_widget(
                    Paragraph::new(format!("{title}\n\n{message}\n\nEsc close"))
                        .style(Style::default().fg(theme.text).bg(theme.background_panel)),
                    inner,
                );
            }
            DialogKind::CommandPalette => {
                crate::widgets::command_palette::render(frame, inner, theme);
            }
            _ => {
                frame.render_widget(
                    Paragraph::new("Dialog")
                        .style(Style::default().fg(theme.text).bg(theme.background_panel)),
                    inner,
                );
            }
        }
    }
}

fn centered_rect(area: Rect, desired_width: u16, desired_height: u16) -> Rect {
    let width = desired_width.min(area.width.saturating_sub(2));
    let height = desired_height.min(area.height.saturating_sub(2));

    let [rect] = Layout::horizontal([Constraint::Length(width)])
        .flex(Flex::Center)
        .areas(area);

    let [rect] = Layout::vertical([Constraint::Length(height)])
        .flex(Flex::Center)
        .areas(rect);

    rect
}

fn panel_height(entry: &DialogEntry, area: Rect) -> u16 {
    match entry.kind {
        DialogKind::CommandPalette => area.height.saturating_sub(6).min(24).max(8),
        DialogKind::Help => area.height.saturating_sub(4).min(28).max(8),
        _ => 10,
    }
}
```

```rust
use ratatui::{
    Frame,
    layout::Rect,
    style::{Color, Style},
    text::{Line, Span},
    widgets::{Block, Borders, Clear, Paragraph},

#[derive(Debug, Clone, Copy)]
pub enum ToastVariant {
    Info,
    Success,
    Warning,
    Error,
}

#[derive(Debug, Clone)]
pub struct Toast {
    pub variant: ToastVariant,
    pub title: Option<String>,
    pub message: String,
    pub created_at: Instant,
    pub duration: Duration,
}

#[derive(Debug, Default)]
pub struct ToastState {
    items: Vec<Toast>,
}

impl ToastState {
    pub fn show(&mut self, variant: ToastVariant, message: impl Into<String>) {
        self.items.push(Toast {
            variant,
            title: None,
            message: message.into(),
            created_at: Instant::now(),
            duration: Duration::from_millis(3_000),
        });
    }

    pub fn retain_live(&mut self) {
        self.items.retain(|toast| toast.created_at.elapsed() < toast.duration);
    }

    pub fn render(&self, frame: &mut Frame, area: Rect, theme: &Theme) {
        let Some(toast) = self.items.last() else {
            return;
        };

        let width = area.width.min(56);
        let height = 5;
        let x = area.x + area.width.saturating_sub(width + 2);
        let y = area.y + 1;

        if width < 10 || area.height < height + 1 {
            return;
        }

        let rect = Rect {
            x,
            y,
            width,
            height,
        };

        let accent = match toast.variant {
            ToastVariant::Info => theme.info,
            ToastVariant::Success => theme.success,
            ToastVariant::Warning => theme.warning,
            ToastVariant::Error => theme.error,
        };

        frame.render_widget(Clear, rect);
        frame.render_widget(
            Block::default()
                .borders(Borders::ALL)
                .border_style(Style::default().fg(accent))
                .style(Style::default().bg(theme.background_panel)),
            rect,
        );

        let inner = rect.inner(ratatui::layout::Margin {
            horizontal: 2,
            vertical: 1,
        });

        let title = toast.title.as_deref().unwrap_or(match toast.variant {
            ToastVariant::Info => "Info",
            ToastVariant::Success => "Success",
            ToastVariant::Warning => "Warning",
            ToastVariant::Error => "Error",
        });

        frame.render_widget(
            Paragraph::new(vec![
                Line::from(Span::styled(title, Style::default().fg(accent))),
                Line::from(Span::styled(&toast.message, Style::default().fg(theme.text))),
            ])
            .style(Style::default().bg(theme.background_panel)),
            inner,
        );
    }
```

## 7.10 Command palette

Current `dialog-command.tsx` is a registry-backed command palette with suggestions, categories, hidden/enabled commands, keybinds, and slash command support.

Rust model:

```rust
// crates/jekko-tui/src/command.rs

use fuzzy_matcher::{skim::SkimMatcherV2, FuzzyMatcher};

#[derive(Debug, Clone)]
pub struct Command {
    pub title: String,
    pub value: String,
    pub description: Option<String>,
    pub category: Option<String>,
    pub keybind: Option<String>,
    pub suggested: bool,
    pub hidden: bool,
    pub enabled: bool,
}

#[derive(Debug, Default)]
pub struct CommandRegistry {
    commands: Vec<Command>,
}

impl CommandRegistry {
    pub fn register(&mut self, command: Command) {
        self.commands.push(command);
    }

    pub fn visible_matches(&self, query: &str) -> Vec<Command> {
        let matcher = SkimMatcherV2::default();

        let mut scored: Vec<(i64, Command)> = self.commands
            .iter()
            .filter(|command| !command.hidden && command.enabled)
            .filter_map(|command| {
                if query.trim().is_empty() {
                    return Some((if command.suggested { 100 } else { 0 }, command.clone()));
                }

                let haystack = format!(
                    "{} {} {}",
                    command.title,
                    command.value,
                    command.description.as_deref().unwrap_or("")
                );

                matcher.fuzzy_match(&haystack, query).map(|score| (score, command.clone()))
            })
            .collect();

        scored.sort_by(|a, b| b.0.cmp(&a.0));
        scored.into_iter().map(|(_, command)| command).collect()
    }
}
```

## 7.11 Prompt / textarea replacement

The current prompt system is complex: prompt history, stash, autocomplete, file/agent parts, textarea keybindings, usage display, submit behavior, and route/session creation.

Use `ratatui-textarea` rather than hand-rolling Unicode cursor editing. Its docs describe it as a Ratatui textarea widget with insertion/deletion, auto-scrolling, undo/redo, line numbers, cursor highlight, soft wrapping, search, selection, mouse scrolling, and multiple textarea support. ([Docs.rs][6])

Ratatui’s own user-input example also cautions that richer input UIs should use `ratatui-textarea` or `tui-input`. ([Ratatui][7])

```rust
// crates/jekko-tui/src/widgets/prompt.rs

use crossterm::event::{KeyCode, KeyEvent, KeyModifiers};
use ratatui::{
    Frame,
    layout::{Constraint, Layout, Rect},
    style::Style,
    text::{Line, Span},
    widgets::{Block, Borders, Paragraph},
};
use ratatui_textarea::{Input, TextArea};

use crate::theme::Theme;

pub enum PromptAction {
    Submit(String),
    None,
}

pub struct PromptState {
    textarea: TextArea<'static>,
    pub mode: PromptMode,
    pub suggestions: Vec<String>,
    pub disabled: bool,
}

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum PromptMode {
    Normal,
    Shell,
}

impl Default for PromptState {
    fn default() -> Self {
        let mut textarea = TextArea::default();
        textarea.set_placeholder_text("Ask Jekko…");
        Self {
            textarea,
            mode: PromptMode::Normal,
            suggestions: Vec::new(),
            disabled: false,
        }
    }
}

impl PromptState {
    pub fn input(&self) -> String {
        self.textarea.lines().join("\n")
    }

    pub fn clear(&mut self) {
        self.textarea = TextArea::default();
        self.textarea.set_placeholder_text("Ask Jekko…");
    }

    pub fn handle_key(&mut self, key: KeyEvent) -> PromptAction {
        if self.disabled {
            return PromptAction::None;
        }

        let submit =
            key.code == KeyCode::Enter &&
            !key.modifiers.contains(KeyModifiers::SHIFT) &&
            !key.modifiers.contains(KeyModifiers::ALT);

        if submit {
            let value = self.input();
            if !value.trim().is_empty() {
                self.clear();
                return PromptAction::Submit(value);
            }
            return PromptAction::None;
        }

        let input: Input = key.into();
        self.textarea.input(input);
        PromptAction::None
    }

    pub fn render(&mut self, frame: &mut Frame, area: Rect, theme: &Theme) {
        let [editor, usage] = Layout::vertical([
            Constraint::Min(3),
            Constraint::Length(1),
        ])
        .areas(area);

        self.textarea.set_style(Style::default().fg(theme.text).bg(theme.background_panel));
        self.textarea.set_cursor_line_style(Style::default().bg(theme.background_element));
        self.textarea.set_block(
            Block::default()
                .borders(Borders::ALL)
                .border_style(Style::default().fg(theme.border))
                .style(Style::default().bg(theme.background_panel)),
        );

        frame.render_widget(&self.textarea, editor);

        frame.render_widget(
            Paragraph::new(Line::from(vec![
                chip(" Enter ", theme),
                Span::raw(" send  "),
                chip(" Shift+Enter ", theme),
                Span::raw(" newline  "),
                chip(" Ctrl+P ", theme),
                Span::raw(" commands"),
            ]))
            .style(Style::default().fg(theme.text_muted).bg(theme.background)),
            usage,
        );
    }
}

fn chip<'a>(label: &'a str, theme: &Theme) -> Span<'a> {
    Span::styled(
        label,
        Style::default()
            .fg(theme.text)
            .bg(theme.background_element),
    )
}
```

Ratatui model:

```rust
// crates/jekko-tui/src/routes/session.rs

use crossterm::event::{KeyCode, KeyEvent};
use ratatui::{
    Frame,
    layout::{Constraint, Layout, Rect},
    style::Style,
    text::{Line, Span},
    widgets::{Block, Borders, Paragraph},
};

use crate::{app::App, router::Router, widgets::prompt::PromptAction};

#[derive(Debug, Default)]
pub struct SessionState {
    pub scroll: u16,
    pub sticky_bottom: bool,
    pub prompt: crate::widgets::prompt::PromptState,
    pub messages: Vec<SessionMessage>,
}

#[derive(Debug, Clone)]
pub enum SessionMessage {
    User(String),
    Assistant(String),
    System(String),
}

impl SessionState {
    pub fn has_active_feed(&self) -> bool {
        !self.messages.is_empty()
    }

    pub fn tick(&mut self) {}

    pub fn handle_key(&mut self, key: KeyEvent, _router: &mut Router) -> color_eyre::eyre::Result<()> {
        match key.code {
            KeyCode::PageUp => {
                self.sticky_bottom = false;
                self.scroll = self.scroll.saturating_add(10);
            }
            KeyCode::PageDown => {
                self.scroll = self.scroll.saturating_sub(10);
                if self.scroll == 0 {
                    self.sticky_bottom = true;
                }
            }
            _ => {
                if let PromptAction::Submit(text) = self.prompt.handle_key(key) {
                    self.messages.push(SessionMessage::User(text));
                    self.sticky_bottom = true;
                    self.scroll = 0;
                }
            }
        }

        Ok(())
    }
}

pub fn render(frame: &mut Frame, area: Rect, app: &mut App, _session_id: &str) {
    let theme = &app.theme.current;

    let [body, prompt] = Layout::vertical([
        Constraint::Min(0),
        Constraint::Length(5),
    ])
    .areas(area);

    let sidebar_width = if body.width >= 130 { 42 } else { 0 };

    if sidebar_width > 0 {
        let [main, sidebar] = Layout::horizontal([
            Constraint::Min(0),
            Constraint::Length(sidebar_width),
        ])
        .areas(body);

        render_messages(frame, main, app);
        render_sidebar(frame, sidebar, app);
    } else {
        render_messages(frame, body, app);
    }

    app.session.prompt.render(frame, prompt, theme);
}

fn render_messages(frame: &mut Frame, area: Rect, app: &App) {
    let theme = &app.theme.current;

    frame.render_widget(
        Block::default().style(Style::default().bg(theme.background)),
        area,
    );

    let visible_height = area.height as usize;
    let mut lines = Vec::new();

    for message in &app.session.messages {
        match message {
            SessionMessage::User(text) => {
                lines.push(Line::from(Span::styled(
                    "You",
                    Style::default().fg(theme.primary),
                )));
                lines.extend(wrap_plain(text, area.width.saturating_sub(4), theme.text));
                lines.push(Line::raw(""));
            }
            SessionMessage::Assistant(text) => {
                lines.push(Line::from(Span::styled(
                    "Jekko",
                    Style::default().fg(theme.accent),
                )));
                lines.extend(wrap_plain(text, area.width.saturating_sub(4), theme.text));
                lines.push(Line::raw(""));
            }
            SessionMessage::System(text) => {
                lines.push(Line::from(Span::styled(
                    text,
                    Style::default().fg(theme.text_muted),
                )));
            }
        }
    }

    let total = lines.len();
    let start = if app.session.sticky_bottom {
        total.saturating_sub(visible_height)
    } else {
        total
            .saturating_sub(visible_height)
            .saturating_sub(app.session.scroll as usize)
    };

    let visible = lines
        .into_iter()
        .skip(start)
        .take(visible_height)
        .collect::<Vec<_>>();

    frame.render_widget(
        Paragraph::new(visible)
            .style(Style::default().fg(theme.text).bg(theme.background)),
        area.inner(ratatui::layout::Margin {
            horizontal: 2,
            vertical: 1,
        }),
    );
}

fn render_sidebar(frame: &mut Frame, area: Rect, app: &App) {
    let theme = &app.theme.current;

    frame.render_widget(
        Block::default()
            .borders(Borders::LEFT)
            .border_style(Style::default().fg(theme.border_subtle))
            .style(Style::default().bg(theme.background_panel)),
        area,
    );

    let inner = area.inner(ratatui::layout::Margin {
        horizontal: 2,
        vertical: 1,
    });

    frame.render_widget(
        Paragraph::new("Workspace\n\nContext\nZyal\nJankurai")
            .style(Style::default().fg(theme.text).bg(theme.background_panel)),
        inner,
    );
}

fn wrap_plain(text: &str, _width: u16, color: ratatui::style::Color) -> Vec<Line<'static>> {
    text.lines()
        .map(|line| Line::from(Span::styled(line.to_string(), Style::default().fg(color))))
        .collect()
}
```


Example `xtask` command dispatcher:

```rust
// xtask/src/main.rs

use std::{env, path::PathBuf, process::Command};

fn main() -> anyhow::Result<()> {
    let cmd = env::args().nth(1).unwrap_or_else(|| "help".to_string());

    match cmd.as_str() {
        "build" => run("cargo", &["build", "--release", "-p", "jekko"]),
        "smoke" => smoke(),
        "print-host-binary-path" => {
            let path = PathBuf::from("target").join("release").join(binary_name());
            println!("{}", path.display());
            Ok(())
        }
        "forbidden-deps" => forbidden_deps(),
        _ => {
            eprintln!("usage: cargo xtask <build|smoke|print-host-binary-path|forbidden-deps>");
            Ok(())
        }
    }
}

fn run(program: &str, args: &[&str]) -> anyhow::Result<()> {
    let status = Command::new(program).args(args).status()?;
    anyhow::ensure!(status.success(), "{program} failed");
    Ok(())
}

fn smoke() -> anyhow::Result<()> {
    run("cargo", &["build", "--release", "-p", "jekko"])?;

    let binary = PathBuf::from("target").join("release").join(binary_name());
    let output = Command::new(binary).arg("--version").output()?;

    anyhow::ensure!(output.status.success(), "jekko --version failed");
    println!("{}", String::from_utf8_lossy(&output.stdout));

    Ok(())
}

fn forbidden_deps() -> anyhow::Result<()> {
    let output = Command::new("git")
        .args(["grep", "-nE", "@opentui|solid-js|opentui-spinner|Bun\\.|bun.lock|bunfig|vite|Vite"])
        .output()?;

    if output.status.success() {
        eprintln!("{}", String::from_utf8_lossy(&output.stdout));
        anyhow::bail!("forbidden OpenTUI/Bun/Vite touchpoints found");
    }

    Ok(())
}

fn binary_name() -> &'static str {
    if cfg!(windows) {
        "jekko.exe"
    } else {
        "jekko"
    }
}
```

# 13. Risk Register

| Risk                      | Why it matters                                                                                                       | Mitigation                                                                                 |
| ------------------------- | -------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------ |
| Plugin API rewrite        | Current plugin API is deeply tied to Solid JSX, OpenTUI renderer, RGBA, ParsedKey, SlotMode, and Bun runtime loading | Freeze external TUI plugins for first Ratatui release; port internal plugins to Rust slots |
| Pixel parity              | OpenTUI flexbox and Ratatui rectangles differ                                                                        | Capture golden screens before deletion; encode layout constants; TestBackend snapshots     |
| Text input complexity     | Prompt has history, stash, parts, autocomplete, multiline, special keybindings                                       | Use `ratatui-textarea`; keep prompt state well-tested                                      |
| Scroll behavior           | OpenTUI scrollbox differs from Ratatui list/paragraph rendering                                                      | Build explicit `ScrollState` and sticky-bottom logic                                       |
| Selection/copy            | OpenTUI has renderer selection APIs; Crossterm mouse capture changes terminal selection behavior                     | Provide config for mouse capture; implement app-level copy where possible                  |
| Syntax highlighting       | Current parser config relies partly on OpenTUI built-ins and parser worker path                                      | Replace with Rust `syntect`, tree-sitter Rust bindings, or plain markdown first            |
| Build artifacts           | Bun produced cross-platform compiled binaries                                                                        | Use Cargo release matrix, `cargo-dist`, `cross`, or `cargo-zigbuild`                       |
| CI/Nix churn              | Node_modules derivation and Bun cache are embedded                                                                   | Replace with Rust cache and `rustPlatform.buildRustPackage`                                |
| Windows terminal behavior | Current `win32.ts` handles Ctrl-C/input quirks                                                                       | Test Crossterm Windows behavior explicitly; add Windows-specific Rust module if needed     |
| Startup performance       | Current app logs first frame and has a startup fallback                                                              | Preserve first-frame watchdog and startup screen in Rust                                   |
