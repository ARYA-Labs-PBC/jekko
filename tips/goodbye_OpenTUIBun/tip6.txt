# Jekko OpenTUI/Bun/Vite → Rust/Ratatui migration engineering plan

## 0. Scope and key finding

I studied the uploaded touchpoints archive: `jekko-open-tui-bun-touchpoints.tar(2).gz`.

The archive appears to be a **touchpoint bundle**, not the entire repository. It contains the important OpenTUI/Bun/TUI files, CI scripts, package manifests, tests, docs, and TUI source, but it does **not** include a Rust workspace or `Cargo.toml`. So the plan below is based on the uploaded files and should be reconciled against the full repo before deleting anything.

### What I found

| Area                                                    | Finding |
| ------------------------------------------------------- | ------: |
| Filtered files in uploaded archive                      |     442 |
| TUI source files under `packages/jekko/src/cli/cmd/tui` |     262 |
| `.tsx` TUI files                                        |     137 |
| `.ts` TUI files                                         |      86 |
| theme `.json` files                                     |      35 |
| `.wav` assets                                           |       4 |
| files with OpenTUI references                           |      96 |
| files with Bun references                               |     140 |
| files with Vite references                              |       0 |
| TUI-related test files                                  |      54 |
| `Cargo.toml` files in uploaded archive                  |       0 |

The practical conclusion is: **this is not a small dependency swap**. The current UI is a Solid/OpenTUI JSX application with providers, slots, dialogs, a plugin system, command registry, prompt editor, session renderer, theme system, and CI/build system deeply tied to Bun/OpenTUI. The clean migration is to replace the entire TUI layer with a Rust crate using Ratatui, while preserving the app’s UX contract, data model, command names, route names, keybindings, themes, and tests.

Ratatui is a Rust TUI library built around terminal backends, widgets, layout areas, and an immediate-mode render loop. Its docs show the canonical path of adding `ratatui` plus `crossterm`, initializing a terminal, drawing frames, and rendering widgets into `Rect` areas. ([Docs.rs][1]) Ratatui’s terminal abstraction maintains buffers and writes only diffs between frames, which is a good fit for recreating the current 60 FPS OpenTUI feel without a DOM/JSX layer. ([Docs.rs][2])

# 1. Recommended target architecture

## 1.1 High-level recommendation

Move the Jekko terminal UI to a **Rust-native immediate-mode application**:

```text
Jekko CLI
└── Rust binary
    ├── Ratatui rendering
    ├── Crossterm terminal/event backend
    ├── Tokio async runtime
    ├── Rust command palette/dialog system
    ├── Rust prompt editor/autocomplete/history/stash
    ├── Rust session timeline/tool renderers
    ├── Rust theme loader
    ├── Rust keymap/action dispatcher
    ├── Rust plugin-slot replacement for internal plugins
    └── Optional future Rust/WASM/data plugin ABI
```

Do **not** try to wrap Ratatui inside TypeScript. That would keep the exact complexity this migration is meant to remove. Also do **not** try to keep Solid/OpenTUI as an adapter layer. The current code relies on OpenTUI renderables, JSX components, Solid providers, and Bun APIs; retaining any of that will keep Bun/OpenTUI alive.

The canonical Ratatui app shape should be:

```text
Input event → App::update(...) mutates state → App::render(...) draws frame
```

Ratatui’s docs use a terminal/backend model, layout constraints, widgets, and stateful widgets, so the cleanest replacement is explicit Rust state plus explicit layout, not a component runtime that imitates JSX. ([Ratatui][3]) For async input and background tasks, use `tokio`, `mpsc` channels, and `crossterm::event::EventStream`; Ratatui’s async-event example demonstrates that exact pattern. ([Ratatui][4])


Recommendation:

1. Port all **internal feature plugins** into compiled Rust modules.
2. Replace the external TUI plugin ABI with either:

   * a Rust plugin API,
   * a WASM plugin API,
   * or a data-only extension API for commands, themes, prompts, and declarative panels.
3. Mark the old TypeScript TUI plugin ABI as removed.

```rust
// crates/jekko-tui/src/terminal.rs

use std::{io, time::Duration};

use anyhow::Result;
use crossterm::{
    event::{Event as CrosstermEvent, EventStream, KeyEventKind},
    execute,
    terminal::{
        disable_raw_mode, enable_raw_mode, EnterAlternateScreen, LeaveAlternateScreen,
    },
};
use futures::{FutureExt, StreamExt};
use ratatui::{backend::CrosstermBackend, Terminal};
use tokio::{sync::mpsc, time};

use crate::{app::App, event::AppEvent};

pub type TuiTerminal = Terminal<CrosstermBackend<io::Stdout>>;

pub struct TerminalGuard;

impl TerminalGuard {
    pub fn enter() -> Result<(Self, TuiTerminal)> {
        enable_raw_mode()?;
        let mut stdout = io::stdout();
        execute!(stdout, EnterAlternateScreen)?;
        let backend = CrosstermBackend::new(stdout);
        let terminal = Terminal::new(backend)?;
        Ok((Self, terminal))
    }
}

impl Drop for TerminalGuard {
    fn drop(&mut self) {
        let _ = disable_raw_mode();
        let _ = execute!(io::stdout(), LeaveAlternateScreen);
    }
}

pub async fn run(mut app: App) -> Result<()> {
    let (_guard, mut terminal) = TerminalGuard::enter()?;

    let (tx, mut rx) = mpsc::channel::<AppEvent>(256);

    spawn_input_reader(tx.clone());
    spawn_tick_loop(tx.clone(), Duration::from_millis(16));

    tx.send(AppEvent::Render).await.ok();

    while let Some(event) = rx.recv().await {
        match app.update(event)? {
            crate::action::Action::Quit => break,
            crate::action::Action::Render | crate::action::Action::None => {}
            action => app.apply_action(action)?,
        }

        terminal.draw(|frame| app.render(frame))?;

        if app.should_quit {
            break;
        }
    }

    Ok(())
}

fn spawn_tick_loop(tx: mpsc::Sender<AppEvent>, every: Duration) {
    tokio::spawn(async move {
        let mut interval = time::interval(every);
        loop {
            interval.tick().await;
            if tx.send(AppEvent::Tick).await.is_err() {
                break;
            }
        }
    });
}

fn spawn_input_reader(tx: mpsc::Sender<AppEvent>) {
    tokio::spawn(async move {
        let mut reader = EventStream::new();

        loop {
            let event = reader.next().fuse().await;
            let Some(Ok(event)) = event else {
                break;
            };

            let app_event = match event {
                CrosstermEvent::Key(key) if key.kind == KeyEventKind::Press => {
                    Some(AppEvent::Key(key))
                }
                CrosstermEvent::Mouse(mouse) => Some(AppEvent::Mouse(mouse)),
                CrosstermEvent::Resize(width, height) => {
                    Some(AppEvent::Resize { width, height })
                }
                _ => None,
            };

            if let Some(event) = app_event {
                if tx.send(event).await.is_err() {
                    break;
                }
            }
        }
    });
}
```

Ratatui provides restoration helpers as well; its docs show `ratatui::restore()` disabling raw mode and leaving the alternate screen. ([Docs.rs][8]) I still recommend a local guard because it makes panic/fatal cleanup explicit and testable.

```rust
// crates/jekko-tui/src/app.rs

use ratatui::{
    layout::{Constraint, Layout},
    Frame,
};

use crate::{
    action::Action,
    event::AppEvent,
    router::Route,
    screens::{home, plugin_missing, session, shell, splash},
    widgets::nav_header::NavHeader,
};

pub struct App {
    pub route: Route,
    pub boot: BootState,
    pub theme: ThemeState,
    pub dialogs: DialogStack,
    pub toasts: ToastState,
    pub keymap: KeyMap,
    pub shell: ShellState,
    pub session: SessionState,
    pub home: HomeState,
    pub should_quit: bool,
    // plus SDK, sync, config, local, project, etc.
}

impl App {
    pub fn update(&mut self, event: AppEvent) -> anyhow::Result<Action> {
        if self.dialogs.is_open() {
            if let Some(action) = self.dialogs.update(&event, &self.keymap)? {
                return Ok(action);
            }
        }

        if let Some(action) = self.handle_global_keybindings(&event)? {
            return Ok(action);
        }

        match &self.route {
            Route::Home => self.home.update(event),
            Route::Shell => self.shell.update(event),
            Route::Session { session_id } => {
                self.session.update(session_id.clone(), event)
            }
            Route::Plugin { id } => {
                self.plugins.update_route(id, event)
            }
        }
    }

    pub fn render(&mut self, frame: &mut Frame) {
        let area = frame.area();

        let [header_area, body_area] = Layout::vertical([
            Constraint::Length(1),
            Constraint::Min(0),
        ])
        .areas(area);

        frame.render_widget(
            NavHeader::new(&self.theme.current, self.nav_header_state()),
            header_area,
        );

        match &self.route {
            Route::Home => home::render(frame, body_area, self),
            Route::Shell => shell::render(frame, body_area, self),
            Route::Session { session_id } => {
                session::render(frame, body_area, self, session_id)
            }
            Route::Plugin { id } => {
                plugin_missing::render(frame, body_area, self, id)
            }
        }

        if self.boot.should_show_splash() {
            splash::render(frame, area, self);
        }
```

```rust
use crossterm::event::{KeyCode, KeyEvent, KeyModifiers};

#[derive(Debug, Clone, PartialEq, Eq, Hash)]
pub enum Binding {
    Engage,
    ShellTabCycle,
    ShellTabCycleBack,
    ShellTabSet(u8),
    ShellLeftToggle,
    ThemeModeToggle,
    FeedScrollPageUp,
    FeedScrollPageDown,
    FeedScrollTop,
    FeedScrollBottom,
    SessionNew,
    SessionResume,
    HelpShow,
}

#[derive(Debug, Clone)]
pub struct KeyMap {
    pub leader_active: bool,
}

impl KeyMap {
    pub fn match_key(&self, key: KeyEvent, text_input_focused: bool) -> Option<Binding> {
        let ctrl = key.modifiers.contains(KeyModifiers::CONTROL);
        let shift = key.modifiers.contains(KeyModifiers::SHIFT);

        match key.code {
            KeyCode::Enter if !text_input_focused => Some(Binding::Engage),
            KeyCode::Char('?') if !text_input_focused => Some(Binding::HelpShow),

            KeyCode::Char('b') if ctrl => Some(Binding::ShellLeftToggle),
            KeyCode::Char('t') if ctrl && shift => Some(Binding::ThemeModeToggle),

            KeyCode::Tab if shift => Some(Binding::ShellTabCycleBack),
            KeyCode::Tab => Some(Binding::ShellTabCycle),

            KeyCode::Char('1') if !text_input_focused => Some(Binding::ShellTabSet(1)),
            KeyCode::Char('2') if !text_input_focused => Some(Binding::ShellTabSet(2)),

            KeyCode::PageUp => Some(Binding::FeedScrollPageUp),
            KeyCode::PageDown => Some(Binding::FeedScrollPageDown),
            KeyCode::Home => Some(Binding::FeedScrollTop),
            KeyCode::End => Some(Binding::FeedScrollBottom),

            KeyCode::Char('n') if ctrl => Some(Binding::SessionNew),
            KeyCode::Char('r') if ctrl => Some(Binding::SessionResume),

            _ => None,
        }
    }
}
```

```rust
impl App {
    fn handle_global_keybindings(
        &mut self,
        event: &AppEvent,
    ) -> anyhow::Result<Option<Action>> {
        let AppEvent::Key(key) = event else {
            return Ok(None);
        };

        let text_input_focused = self.focus.is_text_input();
        let Some(binding) = self.keymap.match_key(*key, text_input_focused) else {
            return Ok(None);
        };

        let action = match binding {
            Binding::HelpShow => Action::OpenDialog(Dialog::Help),
            Binding::ThemeModeToggle => Action::ToggleThemeMode,
            Binding::SessionNew => Action::NewSession,
            Binding::SessionResume => {
                if let Some(id) = self.sync.most_recent_root_session_id() {
                    Action::ResumeSession(id)
                } else {
                    Action::Toast(Toast::info("No previous session to resume"))
                }
            }
            Binding::ShellLeftToggle => {
                self.shell.left_visible = !self.shell.left_visible;
                Action::Render
            }
            Binding::ShellTabCycle => {
                self.shell.cycle_tab(1);
                Action::Render
            }
            Binding::ShellTabCycleBack => {
                self.shell.cycle_tab(-1);
                Action::Render
            }
            Binding::ShellTabSet(n) => {
                self.shell.set_tab_by_number(n);
                Action::Render
            }
            Binding::Engage => {
                if matches!(self.route, Route::Home) {
                    Action::Navigate(Route::Shell)
                } else {
                    Action::None
                }
            }
            Binding::FeedScrollPageUp => {
                self.shell.feed.page_up();
                Action::Render
            }
            Binding::FeedScrollPageDown => {
                self.shell.feed.page_down();
                Action::Render
            }
            Binding::FeedScrollTop => {
                self.shell.feed.scroll_top();
                Action::Render
            }
            Binding::FeedScrollBottom => {
                self.shell.feed.scroll_bottom();
                Action::Render
            }
        };

        Ok(Some(action))
    }
```

```rust
    layout::{Constraint, Layout, Rect},
    widgets::{Block, Borders, Paragraph},
    Frame,
};

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum LeftSize {
    Hidden,
    Shown { width: u16, overlay: bool },
}

pub fn resolve_left(width: u16, visible: bool) -> LeftSize {
    if width < 80 {
        LeftSize::Hidden
    } else if width < 100 {
        if visible {
            LeftSize::Shown {
                width: 28,
                overlay: true,
            }
        } else {
            LeftSize::Hidden
        }
    } else if width < 120 {
        LeftSize::Shown {
            width: 28,
            overlay: false,
        }
    } else if width < 160 {
        LeftSize::Shown {
            width: 38,
            overlay: false,
        }
    } else {
        LeftSize::Shown {
            width: 44,
            overlay: false,
        }
    }
}

pub fn render(frame: &mut Frame, area: Rect, app: &mut crate::app::App) {
    let theme = app.theme.current.clone();
    let left = resolve_left(area.width, app.shell.left_visible);

    let [body_area, footer_area] = Layout::vertical([
        Constraint::Min(0),
        Constraint::Length(3),
    ])
    .areas(area);

    match left {
        LeftSize::Hidden => {
            render_center(frame, body_area, app);
        }
        LeftSize::Shown { width, overlay } => {
            if overlay {
                render_center(frame, body_area, app);

                let left_area = Rect {
                    x: body_area.x,
                    y: body_area.y,
                    width: width.min(body_area.width),
                    height: body_area.height,
                };

                render_left_panel(frame, left_area, app);
            } else {
                let [left_area, center_area] = Layout::horizontal([
                    Constraint::Length(width),
                    Constraint::Min(0),
                ])
                .areas(body_area);

                render_left_panel(frame, left_area, app);
                render_center(frame, center_area, app);
            }
        }
    }

    render_shell_footer(frame, footer_area, app, &theme);
}
```

```rust
    let compact = area.width < 32;
    let active = app.shell.active_pane;
    let theme = &app.theme.current;

                    " ".repeat(label.chars().count())
                },
                if selected {
                    theme.accent_style()
                } else {
                    theme.background_style()
                },
            ));
        }
    }

    let lines = if compact {
        vec![Line::from(title_spans)]
```

## 16.2 Prompt input behavior

```rust
// crates/jekko-tui/src/prompt/textarea.rs

use crossterm::event::{KeyCode, KeyEvent, KeyModifiers};

pub enum PromptOutcome {
    None,
    Submit(String),
    OpenCommands,
    OpenWorkspaceSelect,
    HistoryPrev,
    HistoryNext,
    Cancel,
}

impl PromptState {
    pub fn handle_key(&mut self, key: KeyEvent) -> PromptOutcome {
        let ctrl = key.modifiers.contains(KeyModifiers::CONTROL);
        let shift = key.modifiers.contains(KeyModifiers::SHIFT);

        match key.code {
            KeyCode::Enter if !shift => {
                let text = self.text();
                if text.trim().is_empty() {
                    PromptOutcome::None
                } else {
                    self.clear();
                    PromptOutcome::Submit(text)
                }
            }
            KeyCode::Char('p') if ctrl => PromptOutcome::OpenCommands,
            KeyCode::Char('w') if ctrl => PromptOutcome::OpenWorkspaceSelect,
            KeyCode::Up if self.textarea.lines().len() == 1 => PromptOutcome::HistoryPrev,
            KeyCode::Down if self.textarea.lines().len() == 1 => PromptOutcome::HistoryNext,
            KeyCode::Esc => PromptOutcome::Cancel,
            _ => {
                self.textarea.input(key);
                self.recompute_usage();
                PromptOutcome::None
            }
        }
    }

    pub fn text(&self) -> String {
        self.textarea.lines().join("\n")
    }

    pub fn clear(&mut self) {
        self.textarea = ratatui_textarea::TextArea::default();
        self.parts.clear();
        self.recompute_usage();
    }
}
```

```rust
use ratatui::{
    layout::{Constraint, Layout, Rect},
    widgets::{Block, Borders, Paragraph},
    Frame,

pub fn render(frame: &mut Frame, area: Rect, app: &mut crate::app::App) {
    let theme = &app.theme.current;
    let prompt = &mut app.prompt;

    let text_height = prompt
        .textarea
        .lines()
        .len()
        .clamp(1, prompt.max_height as usize) as u16;

    let [autocomplete_area, input_area, meta_area, status_area] = Layout::vertical([
        Constraint::Length(prompt.autocomplete.height()),
        Constraint::Length(text_height + 2),
        Constraint::Length(1),
        Constraint::Length(1),
    ])
    .areas(area);

    if prompt.autocomplete.visible {
        crate::prompt::autocomplete::render(frame, autocomplete_area, app);
    }

    let block = Block::default()
        .borders(Borders::LEFT)
        .border_style(theme.accent_style())
        .style(theme.panel_style());

    let inner = block.inner(input_area);
    frame.render_widget(block, input_area);
    frame.render_widget(&prompt.textarea, inner);

    render_metadata(frame, meta_area, app);
    render_status_line(frame, status_area, app);
}

fn render_metadata(frame: &mut Frame, area: Rect, app: &crate::app::App) {
    let theme = &app.theme.current;
    let line = ratatui::text::Line::from(vec![
        theme.keycap(&format!(" {} ", app.local.agent_label())),
        " ".into(),
        theme.muted(&app.local.model_label()),
        " ".into(),
        theme.muted(&app.local.variant_label()),
        " ".into(),
        theme.accent(&app.zyal.status_label()),
    ]);

    frame.render_widget(Paragraph::new(line), area);
}

fn render_status_line(frame: &mut Frame, area: Rect, app: &crate::app::App) {
    let theme = &app.theme.current;
    let mut spans = Vec::new();

    if app.prompt.status.is_busy() {
        spans.push(crate::widgets::spinner::span(app.tick, theme));
        spans.push(" ".into());
    }

    spans.push(theme.muted(app.prompt.status.label()));
    spans.push("  ".into());
    spans.push(theme.muted(app.prompt.usage.label()));

    frame.render_widget(Paragraph::new(ratatui::text::Line::from(spans)), area);
}
```

## 16.4 Replacing OpenTUI extmarks

OpenTUI’s text buffer/renderable approach allows extmarks. Ratatui does not have the same concept. Use one of two approaches:

### Preferred approach

Represent prompt text as semantic spans before rendering:

```rust
#[derive(Debug, Clone)]
pub struct PromptPart {
    pub range: std::ops::Range<usize>,
    pub kind: PromptPartKind,
}

#[derive(Debug, Clone)]
pub enum PromptPartKind {
    File,
    Agent,
    Model,
    Workspace,
    Mention,
    Plain,
}

pub fn styled_prompt_line<'a>(
    text: &'a str,
    parts: &[PromptPart],
    theme: &crate::theme::Theme,
) -> ratatui::text::Line<'a> {
    use ratatui::text::Span;

    let mut spans = Vec::new();
    let mut cursor = 0;

    for part in parts {
        if cursor < part.range.start {
            spans.push(Span::raw(&text[cursor..part.range.start]));
        }

        let style = match part.kind {
            PromptPartKind::File => theme.file_ref_style(),
            PromptPartKind::Agent => theme.agent_ref_style(),
            PromptPartKind::Model => theme.model_ref_style(),
            PromptPartKind::Workspace => theme.workspace_ref_style(),
            PromptPartKind::Mention => theme.accent_style(),
            PromptPartKind::Plain => theme.text_style(),
        };

        spans.push(Span::styled(&text[part.range.clone()], style));
        cursor = part.range.end;
    }

    if cursor < text.len() {
        spans.push(Span::raw(&text[cursor..]));
    }

    ratatui::text::Line::from(spans)
}
```

### Alternative approach

Use `ratatui-textarea` for editing and draw an overlay line for metadata/hints only. This is faster to ship, but less visually faithful if current extmarks are central to the prompt UX.

## 17.2 Dialog stack

```rust
pub struct DialogStack {
    stack: Vec<Dialog>,
}

impl DialogStack {
    pub fn open(&mut self, dialog: Dialog) {
        self.stack.push(dialog);
    }

    pub fn replace(&mut self, dialog: Dialog) {
        self.stack.clear();
        self.stack.push(dialog);
    }

    pub fn close(&mut self) {
        self.stack.pop();
    }

    pub fn clear(&mut self) {
        self.stack.clear();
    }

    pub fn is_open(&self) -> bool {
        !self.stack.is_empty()
    }

    pub fn update(
        &mut self,
        event: &crate::event::AppEvent,
        keymap: &crate::keymap::KeyMap,
    ) -> anyhow::Result<Option<crate::action::Action>> {
        let Some(dialog) = self.stack.last_mut() else {
            return Ok(None);
        };

        dialog.update(event, keymap)
    }

    pub fn render(
        &mut self,
        frame: &mut ratatui::Frame,
        area: ratatui::layout::Rect,
        theme: &crate::theme::Theme,
    ) {
        let Some(dialog) = self.stack.last_mut() else {
            return;
        };

        dialog.render(frame, area, theme);
    }
}
```

## 17.3 Generic select dialog

```rust
use ratatui::{
    layout::{Constraint, Layout, Rect},
    widgets::{Block, Borders, Clear, List, ListItem, ListState, Paragraph},
    Frame,
};

pub struct SelectDialog {
    pub title: String,
    pub filter: String,
    pub items: Vec<SelectItem>,
    pub selected: usize,
}

pub struct SelectItem {
    pub label: String,
    pub description: Option<String>,
    pub value: String,
}

impl SelectDialog {
    pub fn render(&mut self, frame: &mut Frame, area: Rect, theme: &crate::theme::Theme) {
        let popup = crate::layout::centered_rect(area, 76, 22);
        frame.render_widget(Clear, popup);

        let block = Block::default()
            .title(self.title.as_str())
            .borders(Borders::ALL)
            .border_style(theme.accent_style())
            .style(theme.panel_style());

        let inner = block.inner(popup);
        frame.render_widget(block, popup);

        let [filter_area, list_area, help_area] = Layout::vertical([
            Constraint::Length(1),
            Constraint::Min(0),
            Constraint::Length(1),
        ])
        .areas(inner);

        frame.render_widget(
            Paragraph::new(format!("› {}", self.filter)).style(theme.text_style()),
            filter_area,
        );

        let items = self.filtered_items().map(|item| {
            let mut line = vec![ratatui::text::Span::styled(
                item.label.clone(),
                theme.text_style(),
            )];

            if let Some(desc) = &item.description {
                line.push("  ".into());
                line.push(ratatui::text::Span::styled(
                    desc.clone(),
                    theme.text_muted_style(),
                ));
            }

            ListItem::new(ratatui::text::Line::from(line))
        });

        let mut state = ListState::default();
        state.select(Some(self.selected));

        frame.render_stateful_widget(
            List::new(items).highlight_symbol("› ").highlight_style(theme.selected_style()),
            list_area,
            &mut state,
        );

        frame.render_widget(
            Paragraph::new("Enter select · Esc close · Type to filter")
                .style(theme.text_muted_style()),
            help_area,
        );
    }
}
```

```rust
// crates/jekko-tui/src/session/body.rs

use ratatui::{
    layout::Rect,
    widgets::{Block, Borders, Paragraph, Scrollbar, ScrollbarOrientation, ScrollbarState},
    Frame,
};

pub fn render(
    frame: &mut Frame,
    area: Rect,
    app: &mut crate::app::App,
    session_id: &str,
) {
    let Some(session) = app.sync.session(session_id) else {
        frame.render_widget(
            Paragraph::new("Session not found").style(app.theme.current.error_style()),
            area,
        );
        return;
    };

    let lines = crate::session::renderers::render_session_lines(session, app);

    let paragraph = Paragraph::new(lines)
        .block(Block::default().borders(Borders::NONE))
        .scroll((app.session.scroll_offset, 0));

    frame.render_widget(paragraph, area);

    let mut scrollbar = ScrollbarState::new(app.session.content_height as usize)
        .position(app.session.scroll_offset as usize);

    frame.render_stateful_widget(
        Scrollbar::new(ScrollbarOrientation::VerticalRight),
        area,
        &mut scrollbar,
    );
}
```

```rust
fn render_shell_tool(tool: &ToolCall, theme: &crate::theme::Theme) -> Vec<Line<'static>> {
    let status = match tool.status {
        ToolStatus::Running => theme.accent("running"),
        ToolStatus::Complete => theme.success("complete"),
        ToolStatus::Failed => theme.error("failed"),
        ToolStatus::WaitingPermission => theme.warning("permission"),

            theme.keycap(" Shell "),
            " ".into(),
            Span::styled(tool.title.clone(), theme.text_style()),
            " ".into(),
            status,
        ]),
        Line::styled(format!("$ {}", tool.command), theme.code_style()),
```

```rust
// crates/jekko-tui/src/theme.rs

use ratatui::{
    style::{Color, Modifier, Style},
    text::Span,
};
use serde::Deserialize;

#[derive(Debug, Clone, Deserialize)]
pub struct ThemeFile {
    pub name: String,
    pub mode: ThemeMode,
    pub colors: ThemeColors,
    pub syntax: Option<serde_json::Value>,
}

#[derive(Debug, Clone, Copy, Deserialize, PartialEq, Eq)]
#[serde(rename_all = "lowercase")]
pub enum ThemeMode {
    Dark,
    Light,
}

#[derive(Debug, Clone, Deserialize)]
pub struct ThemeColors {
    pub background: String,
    pub background_panel: String,
    pub background_element: String,
    pub text: String,
    pub text_muted: String,
    pub accent: String,
    pub border_subtle: String,
    pub error: String,
    pub warning: String,
    pub success: String,
}

#[derive(Debug, Clone)]
pub struct Theme {
    pub name: String,
    pub mode: ThemeMode,
    pub background: Color,
    pub background_panel: Color,
    pub background_element: Color,
    pub text: Color,
    pub text_muted: Color,
    pub accent: Color,
    pub border_subtle: Color,
    pub error: Color,
    pub warning: Color,
    pub success: Color,
}

impl Theme {
    pub fn from_file(file: ThemeFile) -> anyhow::Result<Self> {
        Ok(Self {
            name: file.name,
            mode: file.mode,
            background: parse_hex(&file.colors.background)?,
            background_panel: parse_hex(&file.colors.background_panel)?,
            background_element: parse_hex(&file.colors.background_element)?,
            text: parse_hex(&file.colors.text)?,
            text_muted: parse_hex(&file.colors.text_muted)?,
            accent: parse_hex(&file.colors.accent)?,
            border_subtle: parse_hex(&file.colors.border_subtle)?,
            error: parse_hex(&file.colors.error)?,
            warning: parse_hex(&file.colors.warning)?,
            success: parse_hex(&file.colors.success)?,
        })
    }

    pub fn text_style(&self) -> Style {
        Style::default().fg(self.text).bg(self.background)
    }

    pub fn text_muted_style(&self) -> Style {
        Style::default().fg(self.text_muted).bg(self.background)
    }

    pub fn panel_style(&self) -> Style {
        Style::default().fg(self.text).bg(self.background_panel)
    }

    pub fn accent_style(&self) -> Style {
        Style::default().fg(self.accent)
    }

    pub fn keycap<'a>(&self, text: &'a str) -> Span<'a> {
        Span::styled(
            text,
            Style::default()
                .fg(self.text)
                .bg(self.background_element)
                .add_modifier(Modifier::BOLD),
        )
    }

    pub fn muted<'a>(&self, text: &'a str) -> Span<'a> {
        Span::styled(text, self.text_muted_style())
    }

    pub fn accent<'a>(&self, text: &'a str) -> Span<'a> {
        Span::styled(text, self.accent_style())
    }
}

pub fn parse_hex(input: &str) -> anyhow::Result<Color> {
    let s = input.trim_start_matches('#');
    if s.len() != 6 {
        anyhow::bail!("expected #RRGGBB color, got {input}");
    }

    let r = u8::from_str_radix(&s[0..2], 16)?;
    let g = u8::from_str_radix(&s[2..4], 16)?;
    let b = u8::from_str_radix(&s[4..6], 16)?;

    Ok(Color::Rgb(r, g, b))
}
```

## 20.1 Immediate recommendation

For the first Ratatui release, support **internal slots only**. That gives you the same UI functionality without trying to make TypeScript JSX plugins work in Rust.

Existing slots to preserve as Rust enum values:

```rust

pub enum SlotName {
    App,
    HomeLogo,
    HomePrompt,
    HomePromptRight,
    SessionPrompt,
    SessionPromptRight,
    HomeBottom,
    HomeFooter,
    SidebarTitle,
    SidebarContent,
    SidebarFooter,

    ShellLeftActivePane,
    ShellCenterFeed,
    ShellFooter,
```

Slot registry:

```rust
use ratatui::{layout::Rect, Frame};

pub trait SlotRenderer {
    fn order(&self) -> i32 {
        0
    }

    fn render(
        &mut self,
        frame: &mut Frame,
        area: Rect,
        ctx: &mut SlotContext,
    );
}

pub struct SlotRegistry {
    slots: std::collections::HashMap<SlotName, Vec<Box<dyn SlotRenderer + Send>>>,
}

impl SlotRegistry {
    pub fn register(
        &mut self,
        name: SlotName,
        renderer: impl SlotRenderer + Send + 'static,
    ) {
        self.slots.entry(name).or_default().push(Box::new(renderer));
        self.slots
            .get_mut(&name)
            .unwrap()
            .sort_by_key(|renderer| renderer.order());
    }

    pub fn render_single_winner(
        &mut self,
        name: SlotName,
        frame: &mut Frame,
        area: Rect,
        ctx: &mut SlotContext,
    ) -> bool {
        let Some(renderers) = self.slots.get_mut(&name) else {
            return false;
        };

        let Some(renderer) = renderers.last_mut() else {
            return false;
        };

        renderer.render(frame, area, ctx);
        true
    }

    pub fn render_all(
        &mut self,
        name: SlotName,
        frame: &mut Frame,
        area: Rect,
        ctx: &mut SlotContext,
    ) {
        if let Some(renderers) = self.slots.get_mut(&name) {
            for renderer in renderers {
                renderer.render(frame, area, ctx);
            }
        }
    }
}
```

## 20.2 Plugin decision point

You need a product decision here:

### Option A — remove external TUI plugins for now

Best for a clean Rust migration.

Pros:

```text
No Bun
No JS runtime
No OpenTUI/Solid compatibility burden
Simpler CI/release
Easier pixel-perfect control
```

Cons:

```text
External TUI plugins must be rewritten later
```

### Option B — new data-only plugin ABI

Plugins can provide:

```text
commands
themes
keybindings
declarative menu items
metadata panels
prompt transforms
```

But not arbitrary custom UI drawing.

Pros:

```text
Safer
Can be JSON/TOML/WASM
No JSX
No Bun
```

Cons:

```text
Less powerful than current JSX slot plugins
```

### Option C — WASM UI plugin ABI

Plugins render into a constrained protocol:

```text
Plugin returns a tree of declarative widgets
Host maps widget tree to Ratatui
```

Pros:

```text
Still extensible
Language-neutral
No Bun
```

Cons:

```text
Larger implementation
More protocol work
Harder to make pixel-perfect quickly
```

Recommendation: **Option A for the first Rust release**, then Option B or C later.

# 27. Hardest migration areas

## 27.1 Prompt editor

The prompt is the most complex single component. Current `component/prompt/index.tsx` is over 2,000 lines and combines editing, autocomplete, status, workspace, model/agent/variant state, ZYAL, usage, history, stash, and paste behavior.

Recommendation:

```text
Port prompt early.
Do not leave it until the end.
Do not try to make a perfect custom editor first.
Start with ratatui-textarea, then layer Jekko-specific rendering.
```

## 27.2 Plugin system

The current TUI plugin system is OpenTUI/Solid-specific. This cannot remain if Bun/OpenTUI are gone.

Recommendation:

```text
Treat plugin migration as an ABI break.
Keep command/theme/data plugins if possible.
Remove arbitrary JSX UI plugins from the first Ratatui release.
```

## 27.3 Syntax/rendering workers

Current build copies OpenTUI parser workers. Ratatui replacement should use Rust-native rendering:

```text
pulldown-cmark for markdown
syntect or tree-sitter for code
custom YAML tokenizer port
custom terminal tokenizer port
```

## 27.4 Mouse/copy/select

OpenTUI currently has renderer selection and copy behavior. In Ratatui, you need explicit state:

```rust
pub struct SelectionState {
    pub active: bool,
    pub start: Option<(u16, u16)>,
    pub end: Option<(u16, u16)>,
    pub selected_text: Option<String>,
}
```

Start with keyboard copy and basic clipboard integration; then add mouse selection once core UX is stable.

## 27.5 Windows terminal behavior

Current `win32.ts` uses Bun FFI. Replace with the `windows` crate and keep all Windows terminal setup in one `platform/win32.rs`.

## 27.6 Pixel-perfect layout

OpenTUI has JSX/flex-like layout. Ratatui uses explicit `Rect` layout and constraints. Pixel-perfect means every screen needs snapshot tests at multiple terminal sizes.

Do not rely on eyeballing.

# 30. Bottom-line recommendation

The cleanest migration is:

```text
Delete the OpenTUI/Solid/Bun TUI entirely.
Create a Rust/Ratatui TUI crate.
Port the state model, routes, themes, keybindings, prompt, dialogs, shell, and session renderers.
Compile current internal plugins as Rust modules.
Break the old external JSX TUI plugin ABI.
Add CI guards that make OpenTUI/Bun/Vite impossible to reintroduce.
```

The biggest risks are the prompt editor, plugin ABI, session tool renderers, and pixel-perfect layout. The right way to control those risks is to port them into explicit Rust state machines and back everything with Ratatui golden snapshots plus PTY smoke tests.
