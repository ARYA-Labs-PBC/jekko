# Engineering Work Plan — Replace OpenTUI/Bun/Vite with Rust + Ratatui

## 1. Executive Summary

I inspected the uploaded project archive and found that the current TUI is deeply built around **OpenTUI + Solid JSX + Bun**. The correct migration is not a dependency swap. It is a full UI/runtime rewrite into a Rust terminal application with a canonical Ratatui architecture.

The target end-state should be:

```text
Rust binary
  ├─ Ratatui rendering
  ├─ Crossterm input/events
  ├─ Rust app state/model/update loop
  ├─ Rust prompt/autocomplete/dialog/session widgets
  ├─ Rust API/SSE client
  ├─ Rust config/theme/KV storage
  ├─ Rust plugin/slot registry, or a data-driven plugin format
  ├─ Cargo/xtask build scripts
  ├─ Cargo-based CI/Nix
  └─ zero Bun, zero OpenTUI, zero Solid, zero Vite
```

The most important finding: **there are no Vite references in the uploaded archive**. I found many OpenTUI, Solid, and Bun touchpoints, but no `vite`, `Vite`, or `@vite` hits. So Vite removal is mostly a validation gate unless the full repository has files not present in this touchpoint archive.

Ratatui is the right direction for a 100% Rust TUI, but it requires embracing Ratatui’s immediate-mode model. Ratatui provides widgets and rendering primitives; the application owns state, input handling, routing, async backend events, focus, dialogs, prompt state, and command dispatch. The official docs explicitly describe that Ratatui does not handle input or application state for you, and that event handling is typically done with a backend/input library such as Crossterm. ([Ratatui][1]) Ratatui widgets are composable building blocks, including `Widget` and `StatefulWidget`, rendered through the frame/root-widget pattern. ([Docs.rs][2]) For this project, use `ratatui::init()` / `ratatui::restore()` rather than a tiny `ratatui::run()` wrapper, because Jekko needs mouse support, paste handling, logs/external output behavior, panic cleanup, async streams, and custom terminal recovery; the docs show `init` sets up the Crossterm backend, raw mode, alternate screen, and panic hook, while `restore` returns the terminal from raw/alternate-screen mode. ([Docs.rs][3])

## 3. Non-Negotiable Migration Decisions

### Decision 1 — Do not attempt a JSX-to-Ratatui shim

Do **not** create a Rust macro DSL that imitates:

```tsx
<box>
  <text>
    <span />
  </text>
</box>
```

That will produce a worse architecture than both OpenTUI and Ratatui.

Instead, move to canonical Ratatui:

```text
App state
  ↓
update(event/action)
  ↓
render(frame)
  ↓
widgets render into Buffer
```

Use structs like:

```rust
App
Route
ShellScreen
SessionScreen
PromptState
AutocompleteState
DialogStack
CommandRegistry
Theme
SlotRegistry
```

### Decision 2 — Keep all UI state in Rust

OpenTUI/Solid currently gives you reactive `createSignal`, contexts, `onMount`, and plugin slots. Ratatui does not. Replace those with explicit Rust state.

Example mapping:

```text
Solid Context Provider       -> App substate struct
createSignal                 -> normal field + Action update
createMemo                   -> computed method
onMount                      -> initialization action / async task start
onCleanup                    -> Drop or App shutdown hook
useKeyboard                  -> focused component handle_event()
OpenTUI renderer frame       -> ratatui Frame
JSX tree                     -> render functions/widgets
```

### Decision 3 — Preserve UX through snapshot tests, not through implementation similarity

Pixel closeness should be verified using:

```text
80x24
100x30
120x30
160x45
200x60
light theme
dark theme
loading state
empty shell
active session
prompt focused
autocomplete open
command dialog open
repo-intel pane open
history pane open
permission prompt open
```

The implementation should be idiomatic Rust/Ratatui, not a line-by-line TypeScript port.

### Decision 4 — External TSX TUI plugins cannot remain unchanged

`packages/plugin/src/tui.ts` exposes a TUI plugin API based on:

```text
Solid JSX.Element
@opentui/core CliRenderer
@opentui/core ParsedKey
@opentui/core RGBA
@opentui/solid SolidPlugin
OpenTUI SlotMode
```

That is incompatible with “remove OpenTUI and Bun 100%.”

The clean choices are:

1. **Recommended:** Replace TUI plugins with Rust-native internal plugin modules and a Rust slot registry.
2. **Optional future:** Add a data-driven plugin layer using JSON/TOML/WASM, but not JS/Solid/OpenTUI.
3. **Do not do:** Keep JS TUI plugins running in a Node/Bun sidecar. That violates the goal.

## 5.4 Replace plugin API

Current:

```text
packages/plugin/src/tui.ts
```

Final:

```text
crates/jekko-plugin-api/src/lib.rs
crates/jekko-plugin-api/src/slot.rs
crates/jekko-plugin-api/src/command.rs
crates/jekko-plugin-api/src/theme.rs
```

The old API exposes Solid/OpenTUI types. It must be deleted or rewritten.

Recommended new Rust API:

```rust
pub trait TuiPlugin: Send + Sync + 'static {
    fn id(&self) -> &'static str;
    fn register(&self, api: &mut PluginApi);
}

pub struct PluginApi<'a> {
    pub commands: &'a mut CommandRegistry,
    pub slots: &'a mut SlotRegistry,
}

pub enum SlotMode {
    Replace,
    Before,
    After,
}

pub type SlotRenderFn =
    Box<dyn Fn(&mut ratatui::buffer::Buffer, ratatui::layout::Rect, &RenderContext<'_>) + Send + Sync>;

pub struct SlotRegistry {
    entries: indexmap::IndexMap<String, Vec<SlotEntry>>,
}

pub struct SlotEntry {
    pub plugin_id: String,
    pub order: i32,
    pub mode: SlotMode,
    pub render: SlotRenderFn,
}
```

If external plugins are part of the public product, introduce a new compatibility boundary:

```text
Old: TSX/Solid/OpenTUI runtime plugin
New: JSON/TOML command + slot descriptors, or WASM plugin ABI, or Rust dynamic/static plugin modules
```

Do not run JS TUI plugins if the goal is 100% Bun removal.

# 6. Ratatui Runtime Architecture

## 6.1 Main binary

Create:

```text
crates/jekko-cli/src/main.rs
```

```rust
use color_eyre::Result;
use jekko_tui::{App, RunOptions};

#[tokio::main]
async fn main() -> Result<()> {
    color_eyre::install()?;

    let options = RunOptions::from_env_and_args()?;
    let app = App::new(options).await?;

    jekko_tui::runtime::run(app).await
}
```

## 6.2 Runtime loop

Create:

```text
crates/jekko-tui/src/runtime.rs
```

This replaces the OpenTUI renderer in `app.tsx`.

```rust
use std::{io, time::{Duration, Instant}};

use color_eyre::Result;
use crossterm::{
    event::{
        self, DisableBracketedPaste, DisableMouseCapture, EnableBracketedPaste,
        EnableMouseCapture, Event as CrosstermEvent,
    },
    execute,
};
use ratatui::DefaultTerminal;
use tokio::{select, sync::mpsc, time};
use tracing::error;

use crate::{action::Action, app::App};

pub enum RuntimeEvent {
    Terminal(CrosstermEvent),
    Tick(Instant),
    Backend(crate::state::sdk::BackendEvent),
}

pub async fn run(mut app: App) -> Result<()> {
    let mut terminal = ratatui::init();

    // OpenTUI currently enables mouse except in specific test modes.
    // Do the same here explicitly.
    execute!(
        io::stdout(),
        EnableMouseCapture,
        EnableBracketedPaste,
    )?;

    let result = run_inner(&mut terminal, &mut app).await;

    let cleanup_result = execute!(
        io::stdout(),
        DisableBracketedPaste,
        DisableMouseCapture,
    );

    ratatui::restore();

    if let Err(err) = cleanup_result {
        error!(?err, "failed to restore mouse/paste terminal modes");
    }

    result
}

async fn run_inner(terminal: &mut DefaultTerminal, app: &mut App) -> Result<()> {
    let mut tick = time::interval(Duration::from_millis(16));
    let mut backend_rx = app.start_backend_tasks().await?;

    loop {
        terminal.draw(|frame| {
            app.render(frame);
        })?;

        select! {
            _ = tick.tick() => {
                app.update(Action::Tick(Instant::now()))?;
            }

            maybe_event = backend_rx.recv() => {
                match maybe_event {
                    Some(event) => app.update(Action::Backend(event))?,
                    None => app.update(Action::BackendDisconnected)?,
                }
            }

            terminal_event = read_terminal_event() => {
                app.update(Action::Terminal(terminal_event?))?;
            }
        }

        if app.should_quit() {
            break;
        }
    }

    Ok(())
}

async fn read_terminal_event() -> io::Result<CrosstermEvent> {
    tokio::task::spawn_blocking(event::read)
        .await
        .map_err(|err| io::Error::new(io::ErrorKind::Other, err))?
}
```

```rust
        app.register_builtin_plugins();

    }

    pub fn render(&mut self, frame: &mut Frame<'_>) {

        match &self.route {
            Route::Home => self.home.render(frame, area, self),
            Route::Shell => self.shell.render(frame, area, self),
            Route::Session { session_id, prompt } => {
                self.session.render(frame, area, self, session_id, prompt.as_deref());
            }
            Route::Plugin { id, data } => {
                self.render_plugin_route(frame, area, id, data);
            }

        self.dialogs.render(frame, area, &self.theme);

        if self.dialogs.is_open() {
            if self.dialogs.handle_action(&action, &mut self.commands)? {
                return Ok(());
            }

            Action::Terminal(event) => self.handle_terminal_event(event)?,
            Action::Backend(event) => self.handle_backend_event(event)?,

            Action::Navigate(route) => self.navigate(route),
            Action::Quit => self.should_quit = true,
            Action::BackendDisconnected => self.toasts.warning("backend disconnected"),
        }

        Ok(())
    }

    pub fn should_quit(&self) -> bool {
        self.should_quit
    }
```

# 7. OpenTUI-to-Ratatui UX Mapping

## 7.1 Element mapping

| Current OpenTUI/Solid concept  | Ratatui replacement                             |
| ------------------------------ | ----------------------------------------------- |
| `<box flexDirection="row">`    | `Layout::horizontal([...])`                     |
| `<box flexDirection="column">` | `Layout::vertical([...])`                       |
| `<box position="absolute">`    | render `Clear` + widget into explicit `Rect`    |
| `<text>`                       | `Paragraph`, `Line`, `Span`, or custom `Widget` |
| `<span>`                       | `Span::styled(...)`                             |
| `<scrollbox>`                  | custom viewport widget + `Scrollbar`            |
| `<textarea>`                   | `tui-textarea` plus custom prompt wrapper       |
| `<spinner>`                    | tick-driven spinner widget                      |
| OpenTUI `RGBA`                 | `Color::Rgb`; alpha manually blended            |
| Solid context                  | `App` fields and substate structs               |
| Solid slots                    | Rust `SlotRegistry`                             |
| `useKeyboard`                  | focused component event dispatch                |
| `useMouse`                     | `crossterm::event::MouseEvent`                  |
| `Bun.stringWidth`              | `unicode_width::UnicodeWidthStr`                |
| `Bun.file`, `Bun.write`        | `std::fs`, `tokio::fs`, `serde_json`            |
| `bun:test`                     | Rust unit/integration tests                     |
| `bun run script/*.ts`          | `cargo xtask ...`                               |

## 7.2 Layout model

OpenTUI uses a flexbox-like system. Ratatui does not do CSS/flexbox. Use this pattern everywhere:

```rust
let [header, body, footer] = Layout::vertical([
    Constraint::Length(1),
    Constraint::Min(0),
    Constraint::Length(3),
]).areas(area);

let [left, center] = Layout::horizontal([
    Constraint::Length(left_width),
    Constraint::Min(0),
]).areas(body);
```

For pixel-perfect behavior, centralize responsive calculations in pure functions.

Example:

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum LeftPanelLayout {
    Hidden,
    Inline { width: u16 },
    Overlay { width: u16 },
}

pub fn resolve_left_panel(width: u16, visible: bool) -> LeftPanelLayout {
    if !visible || width < 80 {
        LeftPanelLayout::Hidden
    } else if width < 100 {
        LeftPanelLayout::Overlay { width: 28 }
    } else if width < 120 {
        LeftPanelLayout::Inline { width: 28 }
    } else if width < 160 {
        LeftPanelLayout::Inline { width: 38 }
    } else {
        LeftPanelLayout::Inline { width: 44 }
    }
}
```

This preserves the current `shell-view.tsx` behavior:

```text
width < 80      left panel hidden
width < 100     overlay, width 28
width < 120     inline, width 28
width < 160     inline, width 38
otherwise       inline, width 44
```

```rust
                    x: body.x,
                    y: body.y,
                    width: width.min(body.width),
                    height: body.height,
                };

                frame.render_widget(Clear, left);
                self.render_left_panel(frame, left, app);
            }
        }

        FooterHints::shell().render(frame, footer, &app.theme);

            Constraint::Length(2),
            Constraint::Min(0),
            Constraint::Length(2),

        match self.active_pane {
            ShellPane::RepoIntel => self.repo_intel.render(frame, body, app),
            ShellPane::History => self.history.render(frame, body, app),

            Span::styled("Ctrl+B", app.theme.key_style()),
```

## 9.4 Prompt event handling

```rust
use crossterm::event::{Event, KeyCode, KeyEvent, KeyModifiers};

use crate::{action::Action, app::App};

impl App {
    pub fn handle_prompt_event(&mut self, event: &Event) -> color_eyre::Result<bool> {
        match event {
            Event::Key(key) => self.handle_prompt_key(*key),
            Event::Paste(text) => {
                self.prompt.handle_paste(text)?;
                Ok(true)
            }
            _ => Ok(false),
        }
    }

    fn handle_prompt_key(&mut self, key: KeyEvent) -> color_eyre::Result<bool> {
        if self.prompt.autocomplete.is_open() {
            if self.prompt.autocomplete.handle_key(key, &mut self.prompt)? {
                return Ok(true);
            }
        }

        match (key.modifiers, key.code) {
            (KeyModifiers::CONTROL, KeyCode::Char('p')) => {
                self.commands.show_palette(&mut self.dialogs);
                Ok(true)
            }

            (_, KeyCode::Esc) if self.prompt.shell_mode => {
                self.prompt.shell_mode = false;
                Ok(true)
            }

            (_, KeyCode::Backspace)
                if self.prompt.shell_mode && self.prompt.input_text().is_empty() =>
            {
                self.prompt.shell_mode = false;
                Ok(true)
            }

            (_, KeyCode::Enter) if !key.modifiers.contains(KeyModifiers::SHIFT) => {
                self.submit_prompt()?;
                Ok(true)
            }

            (_, KeyCode::Char('!')) if self.prompt.input_text().is_empty() => {
                self.prompt.shell_mode = true;
                self.prompt.textarea.input(key);
                Ok(true)
            }

            _ => {
                self.prompt.textarea.input(key);
                self.prompt.autocomplete.recompute_from_prompt(&self.prompt);
                Ok(true)
            }
        }
    }
}
```

## 9.5 Paste handling

Current Bun-specific paste and file work should become Rust.

```rust
use std::{path::PathBuf, sync::atomic::{AtomicU64, Ordering}};

static NEXT_PASTE_ID: AtomicU64 = AtomicU64::new(1);

impl PromptState {
    pub fn handle_paste(&mut self, text: &str) -> color_eyre::Result<()> {
        if let Some(path) = detect_single_file_path(text) {
            self.parts.push(classify_file_part(path));
            return Ok(());
        }

        let line_count = text.lines().count();

        if line_count > 8 || text.len() > 2_000 {
            let id = NEXT_PASTE_ID.fetch_add(1, Ordering::Relaxed);
            self.parts.push(PromptPart::PastedText {
                id: format!("paste-{id}"),
                line_count,
                preview: text.lines().take(3).collect::<Vec<_>>().join("\n"),
            });

            self.textarea.insert_str(&format!("[Pasted ~{line_count} lines]"));
        } else {
            self.textarea.insert_str(text);
        }

        Ok(())
    }
}

fn detect_single_file_path(text: &str) -> Option<PathBuf> {
    let trimmed = text.trim();

    if trimmed.contains('\n') {
        return None;
    }

    let path = PathBuf::from(trimmed);

    if path.exists() && path.is_file() {
        Some(path)
    } else {
        None
    }
}

fn classify_file_part(path: PathBuf) -> PromptPart {
    let display = path
        .file_name()
        .and_then(|name| name.to_str())
        .unwrap_or("file")
        .to_string();

    match path.extension().and_then(|ext| ext.to_str()).map(str::to_lowercase) {
        Some(ext) if ["png", "jpg", "jpeg", "gif", "webp"].contains(&ext.as_str()) => {
            PromptPart::Image {
                path: path.display().to_string(),
                display,
            }
        }
        Some(ext) if ext == "pdf" => {
            PromptPart::Pdf {
                path: path.display().to_string(),
                display,
            }
        }
        _ => PromptPart::FileMention {
            path: path.display().to_string(),
            display,
        },
    }
}
```

```rust
use ratatui::{
    Frame,
    layout::Rect,
    style::{Modifier, Style},
    text::{Line, Span},
    widgets::{Block, Borders, Clear, List, ListItem, ListState},
};
use nucleo_matcher::{Matcher, pattern::{CaseMatching, Normalization, Pattern}};

use crate::{app::App, prompt::PromptState};

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum AutocompleteMode {
    Mention,
    Slash,
}

#[derive(Debug, Clone)]
pub struct AutocompleteOption {
    pub id: String,
    pub title: String,
    pub subtitle: Option<String>,
    pub category: Option<String>,
    pub value: String,
}

#[derive(Debug, Default)]
pub struct AutocompleteState {
    pub mode: Option<AutocompleteMode>,
    pub query: String,
    pub selected: usize,
    pub options: Vec<AutocompleteOption>,
}

impl AutocompleteState {
    pub fn is_open(&self) -> bool {
        self.mode.is_some() && !self.options.is_empty()
    }

    pub fn recompute_from_prompt(&mut self, prompt: &PromptState) {
        let text = prompt.input_text();

        if let Some(query) = current_at_query(&text) {
            self.mode = Some(AutocompleteMode::Mention);
            self.query = query.to_string();
            return;
        }

        if let Some(query) = current_slash_query(&text) {
            self.mode = Some(AutocompleteMode::Slash);
            self.query = query.to_string();
            return;
        }

        self.close();
    }

    pub fn close(&mut self) {
        self.mode = None;
        self.query.clear();
        self.selected = 0;
        self.options.clear();
    }

    pub fn filter_options(
        &mut self,
        all: impl IntoIterator<Item = AutocompleteOption>,
    ) {
        let mut matcher = Matcher::new(nucleo_matcher::Config::DEFAULT);
        let pattern = Pattern::parse(
            &self.query,
            CaseMatching::Smart,
            Normalization::Smart,
        );

        let mut scored = Vec::new();

        for option in all {
            let haystack = format!(
                "{} {} {}",
                option.title,
                option.subtitle.as_deref().unwrap_or(""),
                option.category.as_deref().unwrap_or(""),
            );

            if let Some(score) = pattern.score(haystack.as_str(), &mut matcher) {
                scored.push((score, option));
            }
        }

        scored.sort_by(|a, b| b.0.cmp(&a.0));
        self.options = scored.into_iter().map(|(_, option)| option).take(50).collect();
        self.selected = self.selected.min(self.options.len().saturating_sub(1));
    }

    pub fn render_above_prompt(&mut self, frame: &mut Frame<'_>, prompt_area: Rect, app: &App) {
        let height = self.options.len().min(10) as u16 + 2;
        let y = prompt_area.y.saturating_sub(height);

        let area = Rect {
            x: prompt_area.x.saturating_add(2),
            y,
            width: prompt_area.width.saturating_sub(4),
            height: height.min(prompt_area.y),
        };

        if area.height == 0 || area.width == 0 {
            return;
        }

        frame.render_widget(Clear, area);

        let block = Block::new()
            .borders(Borders::ALL)
            .title(match self.mode {
                Some(AutocompleteMode::Mention) => " @ attach ",
                Some(AutocompleteMode::Slash) => " / command ",
                None => "",
            })
            .style(app.theme.popup_style())
            .border_style(app.theme.popup_border_style());

        let items = self.options.iter().map(|option| {
            let mut line = vec![
                Span::styled(option.title.clone(), app.theme.text_style()),
            ];

            if let Some(subtitle) = &option.subtitle {
                line.push(Span::styled(format!("  {subtitle}"), app.theme.muted_style()));
            }

            ListItem::new(Line::from(line))
        });

        let mut list_state = ListState::default();
        list_state.select(Some(self.selected));

        let list = List::new(items)
            .block(block)
            .highlight_style(app.theme.selection_style().add_modifier(Modifier::BOLD))
            .highlight_symbol("› ");

        frame.render_stateful_widget(list, area, &mut list_state);
    }
}

fn current_at_query(text: &str) -> Option<&str> {
    let token = text.split_whitespace().last()?;
    token.strip_prefix('@')
}

fn current_slash_query(text: &str) -> Option<&str> {
    let trimmed = text.trim_start();
    trimmed.strip_prefix('/')
}
```

```rust
pub enum Dialog {
    CommandPalette(crate::command::CommandPaletteState),
    Select(crate::widgets::dialog_select::DialogSelectState),
    Message(crate::widgets::message_dialog::MessageDialogState),
    Confirm(crate::widgets::confirm::ConfirmState),

        self.dialogs.pop();

        !self.dialogs.is_empty()

            return;
        }

            .style(theme.modal_backdrop_style());

        frame.render_widget(overlay, area);

        if let Some(dialog) = self.dialogs.last_mut() {
            let rect = centered_dialog_rect(area, desired_dialog_width(area.width), None);

            frame.render_widget(Clear, rect);

            let block = Block::new()

                .style(theme.dialog_style())

            frame.render_widget(block, rect);

        }

pub fn desired_dialog_width(width: u16) -> u16 {
    if width >= 132 {
        116
    } else if width >= 100 {
        88
    } else {
        60.min(width.saturating_sub(4))
    }
}

pub fn centered_dialog_rect(area: Rect, width: u16, height: Option<u16>) -> Rect {
    let height = height.unwrap_or_else(|| area.height.saturating_mul(2) / 3);

    let vertical = Layout::vertical([
        Constraint::Length(area.height.saturating_sub(height) / 4),
        Constraint::Length(height.min(area.height)),
        Constraint::Min(0),
    ]).split(area);

    let horizontal = Layout::horizontal([
        Constraint::Length(area.width.saturating_sub(width) / 2),
        Constraint::Length(width.min(area.width)),
        Constraint::Min(0),
    ]).split(vertical[1]);

    horizontal[1]
}
```

```rust
use crossterm::event::{KeyCode, KeyEvent};
use ratatui::{
    Frame,
    layout::{Constraint, Layout, Rect},
    text::{Line, Span},
    widgets::{List, ListItem, ListState, Paragraph},
};

use crate::theme::Theme;

#[derive(Debug, Clone)]
pub struct SelectOption {
    pub title: String,
    pub value: String,
    pub description: Option<String>,
    pub category: Option<String>,
    pub keybind: Option<String>,
    pub enabled: bool,
}

#[derive(Debug, Default)]
pub struct DialogSelectState {
    pub title: String,
    pub query: String,
    pub selected: usize,
    pub options: Vec<SelectOption>,
    pub visible: Vec<usize>,
}

impl DialogSelectState {
    pub fn handle_key(&mut self, key: KeyEvent) -> Option<String> {
        match key.code {
            KeyCode::Esc => None,

            KeyCode::Up => {
                self.selected = self.selected.saturating_sub(1);
                None
            }

            KeyCode::Down => {
                self.selected = (self.selected + 1)
                    .min(self.visible.len().saturating_sub(1));
                None
            }

            KeyCode::Home => {
                self.selected = 0;
                None
            }

            KeyCode::End => {
                self.selected = self.visible.len().saturating_sub(1);
                None
            }

            KeyCode::Enter => {
                let index = *self.visible.get(self.selected)?;
                let option = self.options.get(index)?;
                if option.enabled {
                    Some(option.value.clone())
                } else {
                    None
                }
            }

            KeyCode::Backspace => {
                self.query.pop();
                self.refilter();
                None
            }

            KeyCode::Char(ch) => {
                self.query.push(ch);
                self.refilter();
                None
            }

            _ => None,
        }
    }

    pub fn render(&mut self, frame: &mut Frame<'_>, area: Rect, theme: &Theme) {
        let [search, list, footer] = Layout::vertical([
            Constraint::Length(2),
            Constraint::Min(0),
            Constraint::Length(1),
        ]).areas(area);

        frame.render_widget(
            Paragraph::new(Line::from(vec![
                Span::styled("Search ", theme.muted_style()),
                Span::styled(&self.query, theme.input_style()),
            ])),
            search,
        );

        let items = self.visible.iter().filter_map(|&index| {
            let option = self.options.get(index)?;

            let mut spans = vec![
                Span::styled(option.title.clone(), theme.text_style()),
            ];

            if let Some(description) = &option.description {
                spans.push(Span::styled(format!("  {description}"), theme.muted_style()));
            }

            if let Some(keybind) = &option.keybind {
                spans.push(Span::styled(format!("  {keybind}"), theme.key_style()));
            }

            Some(ListItem::new(Line::from(spans)))
        });

        let mut state = ListState::default();
        state.select(Some(self.selected));

        let list_widget = List::new(items)
            .highlight_style(theme.selection_style())
            .highlight_symbol("› ");

        frame.render_stateful_widget(list_widget, list, &mut state);

        frame.render_widget(
            Paragraph::new(Line::from(vec![
                Span::styled("Enter", theme.key_style()),
                Span::raw(" select  "),
                Span::styled("Esc", theme.key_style()),
                Span::raw(" close"),
            ])).style(theme.muted_style()),
            footer,
        );
    }

    fn refilter(&mut self) {
        let query = self.query.to_lowercase();

        self.visible = self.options
            .iter()
            .enumerate()
            .filter(|(_, option)| {
                query.is_empty()
                    || option.title.to_lowercase().contains(&query)
                    || option.description.as_deref().unwrap_or("").to_lowercase().contains(&query)
                    || option.category.as_deref().unwrap_or("").to_lowercase().contains(&query)
            })
            .map(|(index, _)| index)
            .collect();

        self.selected = self.selected.min(self.visible.len().saturating_sub(1));
    }
}
```

```rust
use ratatui::style::Color;
use serde::{Deserialize, Serialize};

#[derive(Debug, Clone, Copy, Serialize, Deserialize, PartialEq, Eq)]
pub struct Rgba {
    pub r: u8,
    pub g: u8,
    pub b: u8,
    #[serde(default = "full_alpha")]
    pub a: u8,
}

fn full_alpha() -> u8 {
    255
}

impl Rgba {
    pub fn to_terminal_color(self, base: Option<Rgba>) -> Color {
        if self.a == 255 {
            Color::Rgb(self.r, self.g, self.b)
        } else {
            let base = base.unwrap_or(Rgba {
                r: 0,
                g: 0,
                b: 0,
                a: 255,
            });

            let blended = blend(base, self);
            Color::Rgb(blended.r, blended.g, blended.b)
        }
    }
}

pub fn blend(base: Rgba, overlay: Rgba) -> Rgba {
    let alpha = overlay.a as f32 / 255.0;
    let inv = 1.0 - alpha;

    Rgba {
        r: ((overlay.r as f32 * alpha) + (base.r as f32 * inv)).round() as u8,
        g: ((overlay.g as f32 * alpha) + (base.g as f32 * inv)).round() as u8,
        b: ((overlay.b as f32 * alpha) + (base.b as f32 * inv)).round() as u8,
        a: 255,
    }
}
```

```rust
        for finding in snapshot.findings.iter().take(6) {
            lines.push(Line::from(vec![
                Span::styled("• ", app.theme.muted_style()),
                Span::styled(&finding.title, app.theme.text_style()),
            ]));

            if let Some(detail) = &finding.detail {
                lines.push(Line::from(vec![
                    Span::raw("  "),
                    Span::styled(truncate(detail, area.width.saturating_sub(2)), app.theme.muted_style()),
                ]));
            }
        }

        frame.render_widget(Paragraph::new(lines), area);
    }
}

fn score_bar(score: u8, width: u16) -> String {
    let cells = if width < 32 { 8 } else if width < 44 { 12 } else { 17 };
    let filled = ((score as usize * cells) + 99) / 100;

    let mut out = String::new();

    for index in 0..cells {
        out.push(if index < filled { '▰' } else { '▱' });
    }

    out
}

fn truncate(text: &str, max_width: u16) -> String {
    use unicode_width::UnicodeWidthStr;

    let max = max_width as usize;

    if UnicodeWidthStr::width(text) <= max {
        return text.to_string();
    }

    let mut out = String::new();

    for ch in text.chars() {
        if UnicodeWidthStr::width(out.as_str()) + UnicodeWidthStr::width(ch.to_string().as_str()) + 1 > max {
            out.push('…');
            break;
        }

        out.push(ch);
    }

    out
}
```

## 14.2 Sticky scroll behavior

```rust
#[derive(Debug, Clone)]
pub struct Viewport {
    content_height: u16,
    offset: u16,
    stick_to_bottom: bool,
}

impl Default for Viewport {
    fn default() -> Self {
        Self {
            content_height: 0,
            offset: 0,
            stick_to_bottom: true,
        }
    }
}

impl Viewport {
    pub fn set_content_height(&mut self, height: u16) {
        self.content_height = height;
    }

    pub fn scroll_up(&mut self, rows: u16) {
        self.offset = self.offset.saturating_sub(rows);
        self.stick_to_bottom = false;
    }

    pub fn scroll_down(&mut self, rows: u16, viewport_height: u16) {
        let max_offset = self.max_offset(viewport_height);
        self.offset = self.offset.saturating_add(rows).min(max_offset);
        self.stick_to_bottom = self.offset == max_offset;
    }

    pub fn stick_to_bottom_if_enabled(&mut self, viewport_height: u16) {
        if self.stick_to_bottom {
            self.offset = self.max_offset(viewport_height);
        }
    }

    pub fn max_offset(&self, viewport_height: u16) -> u16 {
        self.content_height.saturating_sub(viewport_height)
    }
}
```

```rust
use futures::{StreamExt, TryStreamExt};
use reqwest::Client;
use tokio::sync::mpsc;

#[derive(Debug, Clone)]
pub struct ApiClient {
    http: Client,
    base_url: url::Url,
}

#[derive(Debug, Clone, serde::Deserialize)]
pub struct ServerEvent {
    pub kind: String,
    pub data: serde_json::Value,
}

impl ApiClient {
    pub fn new(base_url: url::Url) -> Self {
        Self {
            http: Client::new(),
            base_url,
        }
    }

    pub async fn stream_events(
        &self,
        tx: mpsc::Sender<ServerEvent>,
    ) -> color_eyre::Result<()> {
        let url = self.base_url.join("/events")?;

        let response = self.http
            .get(url)
            .send()
            .await?
            .error_for_status()?;

        let mut stream = eventsource_stream::Eventsource::new(response.bytes_stream());

        while let Some(event) = stream.try_next().await? {
            if event.data.trim().is_empty() {
                continue;
            }

            let parsed: ServerEvent = serde_json::from_str(&event.data)?;
            if tx.send(parsed).await.is_err() {
                break;
            }
        }

        Ok(())
    }
}
```

## 15.3 UI-side batching

```rust
use std::time::{Duration, Instant};

#[derive(Debug)]
pub struct SdkState {
    pending_events: Vec<jekko_api::ServerEvent>,
    last_flush: Instant,
}

impl SdkState {
    pub fn push_event(&mut self, event: jekko_api::ServerEvent) {
        self.pending_events.push(event);
    }

    pub fn should_flush(&self, now: Instant) -> bool {
        now.duration_since(self.last_flush) >= Duration::from_millis(16)
    }

    pub fn flush_into_app(&mut self, app: &mut crate::app::App, now: Instant) {
        let events = std::mem::take(&mut self.pending_events);

        for event in events {
            app.apply_backend_event(event);
        }

        self.last_flush = now;
    }
}
```

```rust
use std::{
    collections::BTreeMap,
    fs::{self, File},
    io::Write,
    path::{Path, PathBuf},
};

use fs2::FileExt;
use serde_json::Value;

#[derive(Debug)]
pub struct KvStore {
    path: PathBuf,
    values: BTreeMap<String, Value>,
}

impl KvStore {
    pub fn open(path: PathBuf) -> color_eyre::Result<Self> {
        let values = if path.exists() {
            let file = File::open(&path)?;
            file.lock_shared()?;
            let values = serde_json::from_reader(&file).unwrap_or_default();
            file.unlock()?;
            values
        } else {
            BTreeMap::new()
        };

        Ok(Self { path, values })
    }

    pub fn get<T: serde::de::DeserializeOwned>(&self, key: &str) -> Option<T> {
        self.values
            .get(key)
            .and_then(|value| serde_json::from_value(value.clone()).ok())
    }

    pub fn set<T: serde::Serialize>(&mut self, key: &str, value: T) -> color_eyre::Result<()> {
        self.values.insert(key.to_string(), serde_json::to_value(value)?);
        self.flush()
    }

    pub fn flush(&self) -> color_eyre::Result<()> {
        if let Some(parent) = self.path.parent() {
            fs::create_dir_all(parent)?;
        }

        let lock_path = self.path.with_extension("lock");
        let lock = File::create(lock_path)?;
        lock.lock_exclusive()?;

        let tmp_path = self.path.with_extension("json.tmp");
        {
            let mut tmp = File::create(&tmp_path)?;
            serde_json::to_writer_pretty(&mut tmp, &self.values)?;
            tmp.write_all(b"\n")?;
            tmp.sync_all()?;
        }

        fs::rename(tmp_path, &self.path)?;

        lock.unlock()?;
        Ok(())
    }
}
```

```rust
use indexmap::IndexMap;
use ratatui::{buffer::Buffer, layout::Rect};

use crate::{app::App, theme::Theme};

pub struct RenderContext<'a> {
    pub app: &'a App,
    pub theme: &'a Theme,
}

pub type SlotRenderer =
    Box<dyn Fn(Rect, &mut Buffer, &RenderContext<'_>) + Send + Sync>;

pub struct SlotEntry {
    pub id: String,
    pub order: i32,
    pub renderer: SlotRenderer,
}

#[derive(Default)]
pub struct SlotRegistry {
    slots: IndexMap<String, Vec<SlotEntry>>,
}

impl SlotRegistry {
    pub fn register(&mut self, slot: impl Into<String>, entry: SlotEntry) {
        let entries = self.slots.entry(slot.into()).or_default();
        entries.push(entry);
        entries.sort_by_key(|entry| entry.order);
    }

    pub fn render_slot(
        &self,
        slot: &str,
        area: Rect,
        buffer: &mut Buffer,
        ctx: &RenderContext<'_>,
    ) {
        if let Some(entries) = self.slots.get(slot) {
            for entry in entries {
                (entry.renderer)(area, buffer, ctx);
            }
        }
    }
}
```

# 22. Hardest Parts and Recommended Solutions

## 22.1 Prompt extmarks / virtual chips

OpenTUI currently supports visual prompt parts and extmarks. Ratatui does not directly provide an equivalent. Recommended solution:

```text
Use tui-textarea for editing.
Maintain a separate PromptPart list.
Render chips/status around the textarea.
For inline chips, build a custom prompt renderer once textarea behavior is stable.
```

Do not start by writing a custom editor from scratch. First ship with `tui-textarea`, then add pixel-perfect chip rendering.

## 22.2 Solid plugin slots

Solid/OpenTUI slots need a clean break. Replace with Rust slot registry.

This will likely require a major-version change for the plugin API.

## 22.3 RGBA alpha

Terminals do not give you arbitrary composited transparency. Blend alpha colors into the known background color before converting to `Color::Rgb`.

## 22.4 Mouse selection / copy behavior

OpenTUI has renderer-level selection/copy behavior. In Ratatui, preserve:

```text
mouse enabled
click rows
click dialogs
click autocomplete
scroll wheel
```

But terminal-native text selection and copy behavior will differ by terminal. Document that and test in:

```text
iTerm2
Apple Terminal
Ghostty
WezTerm
Kitty
Alacritty
VS Code terminal
```

## 22.5 External output passthrough

Current OpenTUI config uses:

```text
externalOutputMode: passthrough
```

In Ratatui, the safest pattern is:

```rust
ratatui::restore();
println!("external output...");
let mut terminal = ratatui::init();
```

For short logs, prefer an in-app log/toast panel. For long external output, temporarily suspend the TUI.

# 25. Final Recommendation

Move all TUI code into a Rust Ratatui workspace and treat the current OpenTUI/Solid/Bun implementation as the behavioral specification, not as code to preserve. The highest-leverage path is:

```text
1. Rust runtime first
2. State/theme/route foundations
3. Shell/home/dialogs
4. Prompt/autocomplete
5. Session transcript
6. API/SSE
7. Built-in panes/plugins
8. Delete OpenTUI/Solid/Bun
9. Lock with snapshots and grep gates
```

The biggest architectural decision is the plugin API. Because the current plugin surface exposes Solid JSX and OpenTUI renderer types, it cannot remain compatible with a 100% Rust/Ratatui system. Replace it with Rust-native slots and commands, and consider a future data-driven or WASM plugin model only after the core Ratatui UI is stable.
